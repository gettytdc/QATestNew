Process Studio/Object Studio

Validation

Overview

The 'Process Validation' tool tests for a number of potential errors in your Blue Prism Process or Visual Business Object. Using this tool will help you detect problems in your work before you make it available for use in Control Room.

Detail

The following paragraphs give a more detailed examination of process/object validation and the concepts used within the interface to aid in identifying and resolving any process issues

Types of result

Validation results can be one of three types:

Error
Errors are generally used to indicate problems which must be fixed in order to avoid your process / business object failing at runtime.
Warning
Warnings alert you to potential problems which may cause your process / business object to fail at runtime. You must use your discretion in deciding whether or not to ignore the error.
Advice
Advice can often give you useful information on how to make the process / business object easier to understand or improve the documentation, but does generally not include problems that will cause any failure during running.
Sorting Results

Results may be sorted in order to group the results by Page, Stage Name, Type, etc. Simply click the header of interest (eg Stage Name) in order to sort according to that attribute (ie sort by Stage Name, in order). Clicking the same column again will toggle the sort order (eg from alphabetic to reverse alphabetic in the Stage Name column).

Getting Help

Sometimes background knowledge is available for a particular message. In this case, the Related Help ... button will be enabled. Clicking this button will take you to the relevant help page.

Alternatively you may contact Blue Prism for support.

Automatic Repair

Results are classed in the list as repairable and non-repairable. Non-repairable indicates that the error must be fixed by a human; repairable indicates that Blue Prism can automatically fix the error.

In the event that one or more of the errors is repairable, the Repair All button will be enabled. On clicking the button, all possible errors which can be repaired (as indicated in the Repairable column) will be repaired.

Inclusion of Business Objects

When viewing a process which uses business objects which have not yet been installed, a large number of results will clutter the list with errors about missing business objects. To exclude these errors, remove the checkmark from the box labelled Include Business Objects.

Go To Stage

The Go To Stage button takes you to the relevant stage, if such a stage exists. For example, if a stage is missing an out-bound link, the button will take you to the stage which is missing a link. If the error is not related to a stage (eg a page exists which is never used), then the button will have no effect.
