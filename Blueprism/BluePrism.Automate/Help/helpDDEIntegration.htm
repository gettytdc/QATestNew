Help - DDE Integration

Overview

Dynamic Data Exchange is a Microsoft technology supported by a number of legacy applications. This mechanism has existed since Windows version 3.1.

DDE provides an additional mechanism for reading and writing data to target applications, as an alternative to interfacing with the presentation layer.

Usage

DDE data fields are identified using three strings: the server name, the topic name and the item name. It is these three pieces of information that make up the attributes of a DDE Element.

To create a DDE Element, right-click an existing element in the Application Modeller tree, and choose "Insert new DDE Element". You will then have to populate the three attributes according to your target application's definitions for the item of interest.

To see a list of available DDE servers, topics and items, right-click the Application Modeller tree and choose "View list of DDE items".

Data Types

All data sent to/from DDE by Blue Prism is treated as text. You may override how data is treated (once read from your target application) by setting a new data type against the element in Application Modeller. Valid options are "Date", "DateTime", "Flag", "Number", "Text", "Time" and "TimeSpan".
