凭据详细信息

“凭据详细信息”窗体允许您编辑新凭据的名称和描述、添加应用程序凭据和设置访问权限。

名称

唯一标识凭据的名称—这是流程在运行时用于查找凭据详细信息的名称。

描述

凭据的任何附加描述性信息。

类型

要添加的凭据的类型。

“应用程序凭据”选项卡

凭据类型

不同的凭据类型反映了支持的身份验证方法。可以使用所选类型的相应字段。

常规
使用此凭据类型进行非 Web API 身份验证。

用户名

通常用于存储登录用户名。

密码

要安全地存储在凭据中的密码。
基本身份验证
使用此凭据类型进行基本的 Web 身份验证以创建身份验证标头。

用户名

授权用户名。

密码

要安全地存储在凭据中的授权密码。
OAuth 2.0（客户端凭据）
使用此凭据类型根据客户端凭据进行 OAuth 2.0 Web 身份验证。

客户端 ID

发出请求的客户端。

客户端密码

用于验证请求。
OAuth 2.0（JWT 持有者令牌）
使用此凭据根据 JSON Web 令牌 (JWT) 进行 OAuth 2.0 Web 身份验证。

颁发者

用于形成身份验证令牌请求。

私钥

客户端密码，用于签署令牌请求。
持有者令牌
使用此凭据类型进行持有者令牌身份验证。

令牌

要在身份验证标头中发送的持有者令牌。

到期日期

凭据到期的日期（如已知）。如该日期已过，则流程将无法使用这些凭据。

标记为无效

如果流程不使用这些凭据，则可以将它们标记为无效。

其他属性

可以为凭据创建任意数量的通用命名属性，这些属性的值将被安全地保存在数据库中。流程可以请求这些属性值。

“访问权限”选项卡

安全角色（推荐的配置）

Blue Prism 帐户为了能够访问凭据所需的安全角色。

这只适用于此类运行时资源：非公共（即 /public）和配置为在用户的上下文（即/SSO 或 /user [username] [ password]）下运行。

按安全角色限制对凭据的访问将阻止在以下情况下访问凭据：

  • 调度程序创建流程会话时
  • 在配置为公共的资源上创建流程会话时

流程（不推荐）

允许访问凭据的流程。

请求凭据时提供的会话标识符必须与活动会话相关，并且会话记录中的流程必须位于允许的流程列表中。

资源（不推荐）

在线时允许访问凭据的资源。

请求凭据时提供的会话标识符必须与活动会话相关，并且会话记录中的资源必须在线并在允许的资源列表中。
