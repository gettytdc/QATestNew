帮助—DDE 集成

概述

数据动态交换是很多传统应用程序都支持的一种 Microsoft 技术。这种机制自 Windows V3.1 起就已被采用。

DDE 提供了另一种针对目标应用程序读取和写入数据的机制，因而无需再与表示层对接。

使用

DDE 数据字段可以通过三个字符串来识别：服务器名称、主题名称和项名称。这三项信息构成了 DDE 元素的三个属性。

要创建 DDE 元素，请右键点击应用程序建模器树中的现有元素，然后选择“插入新 DDE 元素”。然后，您必须根据想要插入的项在目标应用程序中的定义来填写这三个属性。

要查看列有可用 DDE 服务器、主题和项的列表，请右键点击应用程序建模器树，然后选择“查看 DDE 项列表”。

数据类型

发送至和发送自 Blue Prism 的所有数据都会被视为文本。可以通过在应用程序建模器中为元素设置新的数据类型，来覆盖现有的数据处理方式（从目标应用程序读取之后）。 有效的选项为“日期”、“日期选项”、“标志”、“数字”、“文本”、“时间”和“时间间隔”。
