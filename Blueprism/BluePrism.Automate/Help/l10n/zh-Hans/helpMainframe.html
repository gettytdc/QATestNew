大型机集成

简介

在进行大型机集成后，您即可与各种基于终端的应用程序进行交互。这些应用程序通常都是传统的“绿屏”应用程序，以往会通过“哑终端”来访问。这些应用程序会在大型机上运行。在现代化系统上，哑终端已被终端仿真程序取代，后者可以提供各种强大的自动化功能。

大型机向导

Blueprism 会通过各种特定于仿真程序的 COM API 或通过高级语言 API (HLLAPI) 与各种终端仿真程序集成。Blueprism 当前支持以下仿真程序：

  • Attachmate
  • Attachmate Reflection
  • Hummingbird
  • IBM Host on Demand
  • IBM iAccess
  • IBM Personal Communicator
  • Infoconnect
  • NeoWare/WallData Rumba
  • Passport
  • Pericom/HP Teemtalk*
  • PowerTerm WebConnect 5.7

* Teemtalk v4.3 只能支持 3270 个终端。HP Teemtalk v7.0 不支持 DDE 或 HLLAPI 集成，因此需要采用其他技术（如表面自动化）来实现 UI 自动化

通用 HLLAPI 功能或许能支持其他终端。

要与大型机应用程序集成，请运行应用程序建模器，然后选择“大型机”。请参考大型机应用程序参数帮助，以进一步了解如何设置大型机应用程序

探测

加载终端后，需要使用探测工具将终端窗口的具体位置告知 Blueprism。选好终端仿真程序窗口后，Blueprism 将向用户显示一个网格，以用于将这个区域划分成正确的行数和列数（例如，80x24）。之后，便可通过拖移鼠标来识别各个字段，即屏幕上的各个矩形区块。最后，要将这些字段捕获到应用程序管理器中，请将鼠标指针悬停在所需的字段上，然后按住 Ctrl 键并点击鼠标左键。

有关大型机应用程序的更多帮助，请参阅故障排除指南。

向大型机发送按键

“向大型机发送按键”功能可以获取一个由字母和控制代码组成的序列。例如，如果您想发送按键“u”、“s”、“e”、“r”、“ENTER”，则可以使用以下序列：“user{enter}”。“向大型机发送按键”适用于 Micro Focus Reflection .NET 连接器和 Micro Focus Reflection HLLAPI 仿真程序。

加号“+”、插入符号“^”、百分比符号“%”、波浪符号“~”、圆括号“(”和“)”对于 SendKeys 而言具有特殊含义。要指定其中的任一字符，请使用大括号将它括起来。例如，要指定加号，请使用“{+}”。要指定两个大括号字符，请使用“{{}”和“{}}”。方括号“[”和“]”对于 SendKeys 而言没有特殊含义，但必须使用大括号将它们扩起来。
