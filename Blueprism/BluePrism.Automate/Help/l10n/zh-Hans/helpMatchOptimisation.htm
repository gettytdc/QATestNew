应用程序建模器

匹配优化

简介

如果应用程序拥有一个包含很多用户界面元素的复杂用户界面，那么 Blue Prism 可能需要较长的时间才能在应用程序模型中找到相应的元素。很多属性都会对搜索的执行方式产生影响，有时还能加快搜索速度。.

优化属性

父级计数
“父级计数”属性对于包含 Active Accessibility 和 Java 的对象模型等结构树来说尤其有用。在此等模型中，如果我们想要比父级计数更为深入的向下搜索，则可以中途停止沿着树的分支进行的搜索。父级计数对于所有类型的应用程序都非常有用，但仅会为 AA 和 Java 提供优化。父级计数可能会随着模型的更新而改变，所以在使用时要多加小心。“HTML 父级计数”属性会因 IE 版本而异，所以在升级 IE 后必须对使用父级计数的元素重新进行探测。
匹配索引
使用“匹配索引”属性可以使搜索在找到第 N 个元素时立即停止。由于搜索通常会继续查找重复匹配，从而消除模糊匹配，因此在使用此属性时应该十分小心。另外，系统不一定会按照同一顺序查找元素，所有找到的第 N 个元素可能会有别于预期。
反向匹配
使用“匹配反向”属性仅会更改搜索的方向。如果元素位于搜索列表末尾，则该属性非常有用。只有在与“匹配索引”属性结合使用时，“反向匹配”属性才可用。

优化基于浏览器的应用程序

路径

在匹配基于浏览器的应用程序中的某个元素的“路径”属性时，匹配执行速度非常快。如果“路径”属性的值不会变化，则最适合使用这个属性来进行匹配。

元素 ID

如果不适合使用“路径”属性来进行匹配，那么接下来要采用的最佳替代方案应该是匹配元素 ID。元素 ID 是指网页开发人员可能已向元素添加的名称。元素 ID 必须是唯一的。网页中的脚本还可以根据元素 ID 快速找到相应的元素。

排除 HTC 组件

应用程序建模器向导中有一个可用来排除 HTC 组件的选项。启用该选项后，性能将会有所提升。仅当所交互的应用程序使用了 HTC 组件时，才应关闭该选项。

仅与浏览器当前选项卡上的元素进行交互

应用程序建模向导中有一个选项可将交互对象限制为当前选项卡上的元素。该选项可以提供更加一致的结果。系统会默认启用该选项。如果启用了该选项，那么在与其他选项卡进行交互之前，必须先切换到相应的选项卡。关闭该选项后，即可与当前未处于活动状态的选项卡上的元素进行交互；但是，这么做有时可能会产生意料之外的结果。
