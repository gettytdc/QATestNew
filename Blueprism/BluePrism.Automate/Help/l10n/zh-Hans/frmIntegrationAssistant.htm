Object Studio

应用程序建模器

简介

Object Studio 旨在帮助您创建用于与最终用户应用程序交互的小型模块化单元。传统上，为了与此类应用程序进行通信，需要寻求计算机程序员的帮助。Object Studio 为业务用户和技术专家提供了这个机会。

为此，Object Studio 引入了应用程序建模器。应用程序建模器允许您并排运行 Blue Prism 和目标应用程序，并邀请您通过指向和点击教会 Object Studio 掌握目标应用程序的功能。

教学流程—创建新的应用程序元素

通过打开应用程序建模器并点击“添加元素”，您可以将表示用户界面元素的新元素添加到应用程序模型中。通过执行此操作，您可以将 Object Studio 引入到目标应用程序的新功能中。此类功能被称为应用程序元素。举例来说，我们假设您正在向目标应用程序的主窗口引入“确定”按钮。

接下来，您应为新应用程序元素指定一个易记的名称。您可以随意选择任何名称，但“主窗口确定按钮”这个名称可能是最合适的。

下一步是点击“探测详细信息”按钮。执行此操作后，您将有机会向应用程序建模器展示您讨论的目标应用程序中的应用程序元素。从本质上讲，这将仅包括使目标应用程序可见（请记住，该目标应用程序将在此环节运行），导航到相关按钮，并点击此按钮。

在大多数情况下，Blue Prism 将能够检测出您使用探测工具识别的应用程序元素是一个按钮。在这种情况下，标有“元素类型”的组合框将由“按钮”选项填充。在极少数情况下，Blue Prism 无法自动检测到这一事实，因此您必须通过从下拉列表中选择最合适的选项来填充它。

在某些情况下，您可能希望将数据类型与您已识别的应用程序元素相关联。如果是按钮，这么做是不合适的；但如果是表示数字或日期的数据字段，您当然会希望从下拉列表中选择相应的数据类型。这使 Object Studio 可以即场智能地处理数据，从而让您无需手动进行处理。

细化数据—属性

在此环节，您已经识别了相关应用程序元素，并且 Object Studio 已经收集了许多有关该应用程序元素的内部信息。其中一些信息在“详细信息”列表中显示。示例包括按钮上的文本（相关应用程序元素确实是按钮时）、应用程序元素的位置及其所属的窗口等。这些信息逐行显示，每行代表一条信息。这种信息被称为应用程序元素的属性。

Object Studio 稍后使用捕获的属性来识别相同的应用程序元素，特别是在针对目标应用程序运行业务对象时。您现在有机会在后面的环节中帮助 Object Studio 识别应用程序元素。

例如，Object Studio 可能已经观察到您的按钮含有文字“确定”，但是在深入了解目标应用程序后，您会意识到有时相同的按钮会视情况含有文字“保存”。如果不希望 Object Studio 因认为按钮始终含有文字“确定”而被误导，您应该取消勾选文本元素旁边的“正在使用”框。通过这样做，您指示 Object Studio 忽略按钮上的文字，并改为使用其他属性来识别应用程序元素。从这个意义上说，按钮上的文字不再是“正在使用”。

进一步细化—属性匹配类型

Object Studio 知道某些属性会随环境而变，但它们通常随明确的模式发生变化。例如，字处理软件包的窗口标题可能是“我的字处理器—文档 1”，其中“文档 1”是当前正在编辑的文档的名称。标题的“我的字处理器”部分始终是固定的，因此有助于识别窗口。

为此，属性可与表达式而不是固定值进行匹配。表达式由匹配类型确定，可用的匹配类型如下：

等于
完全匹配给定值。
不等于
仅在不是给定值时匹配。
小于
在小于给定值时匹配。
大于
在大于给定值时匹配。
小于或等于
在小于或等于给定值时匹配。
大于或等于
在大于或等于给定值时匹配。
通配符
匹配通配符表达式，其中 * 表示任何字符或没有字符，? 匹配任何单个字符，# 匹配任何单个数字。
正则表达式
匹配正则表达式。
范围
在属于特定的值范围（含上下限值）时匹配。值的语法必须是 X..Y 形式，即下限 (X) 值和上限 (Y) 值由两个点分隔开。
动态
在运行时匹配“读取”、“写入”、“导航”和“等待”环节中的表达式。

根据属性值的数据类型，可以使用不同的匹配类型。例如，“通配符”和“正则表达式”可用于基于文本的属性，而“大于”和“小于”可用于基于数字的属性。还有一种特殊的动态匹配类型可用。动态意味着在设计业务对象时不会立即为属性提供特定值，而是在业务对象运行时提供该值。例如，如果您在运行业务对象的过程中使用字处理器打开名为“2006 年纳税申报表”的文档，则您可以将该文档名提供给参数，以便属性可以使用它。

要更改属性的匹配类型，请从“匹配类型”列的下拉框中选择匹配类型。然后，属性值将与所选的条件匹配。选择“动态”将在应用程序建模器中禁用属性的值。您可以在引用该元素的单个读取、写入、导航和等待属性窗体中设置动态属性。在属性窗体中，您将注意到“...”按钮已启用，按此按钮将允许您输入各种属性的表达式。

元素掩码

您可能会发现您希望应用程序模型中的许多元素使用相同的属性（例如，类名、窗口文本、X 和 Y）。为了实现这一点，您可以创建带有所需属性的元素，复制其“元素掩码”，然后将该掩码粘贴到其他元素上。这将在目标元素中启用已在源元素中启用的相同属性，并将匹配类型（等于、大于等）设置为相同类型。请注意，此操作不会设置属性的值—设置的只是“正在使用”/“匹配类型”配置。

要复制属性掩码，请右键点击源元素并选择“复制元素蒙版”。

要粘贴元素掩码，请右键点击目标元素并选择“粘贴元素蒙版”。

构建您的应用程序—组

您会注意到树视图下方有两个按钮。一个带有文字“添加元素”，而另一个带有文字“添加子元素”。如果点击其中的第二个按钮，效果将是像以前一样创建一个新的应用程序元素，但该新元素将被分组到当前所选元素的下面。从这个意义上讲，新元素被认为是第一个元素的子元素。

此元素分组功能使您能够表示目标应用程序的结构。例如，如果目标应用程序包含多个不同的屏幕，则您可能需要三个或四个“确定”按钮，即每个屏幕一个。通过将每个按钮分组到树中的父窗口下，您可以将这些按钮与其各自的父项相关联。这将有助于在后面的环节中（例如，在编辑读取环节属性时）搜索应用程序元素的树。

请注意，此类分组与 Blue Prism 无关，不会影响业务对象的行为。应用程序元素分组工具只是允许您在业务对象中表示您自己对目标应用程序的认知图。您无需使结构反映目标应用程序的真实布局或其内部工作方式。

调整组—删除、复制和移动元素

应用程序建模器树视图允许您重新组织应用程序元素。要删除元素，请右键点击该元素，然后从上下文菜单中选择适当的选项。请注意，如果删除包含子元素的元素，则其所有子元素也将被删除。如果您想留下此类子元素，您应该先移动它们。

要移动元素，您只需拖动该元素并将其放入新位置即可。如果您将一个元素放在另一个元素的顶部，则放置的元素将成为另一个元素的子元素。如果将一个元素放在两个其他元素之间，则该元素将位于这两个元素之间。

通过在拖放操作期间按住 Shift 键，可将拖动的应用程序元素的副本放置在结束位置，而不是移动它。

筛选应用程序模型

您可以使用应用程序模型树视图上方的筛选器框来筛选该视图。如果输入任何文本，系统将扫描树视图，并且任何与指定筛选器匹配的元素将保持可见，同时系统将隐藏任何不匹配的元素。

筛选不区分大小写，而且，如果元素的名称、描述或任何活动属性值包含筛选器文本，则该元素将与筛选器匹配。不支持通配符或正则表达式。

筛选树时，树处于只读状态，即您仍可以更改属性选择/值，但无法修改应用程序模型的结构。

定位元素

在定义了元素并在 Object Studio 环节（例如，读取、写入或导航环节）中使用这些元素之后，您可能希望找到对特定元素的所有引用。

为此，搜索功能接受用于搜索应用程序元素的特殊语法。
