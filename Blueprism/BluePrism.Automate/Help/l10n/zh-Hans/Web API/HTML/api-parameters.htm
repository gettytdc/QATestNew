参数

通过使用参数，可以在从流程中调用 API 时以动态方式添加值，借此将数据传递到 API 请求。参数可用在基本 URL、操作 URL 路径、标头、请求正文中，并且在向流程公开时，如果将适当的 API 服务选择为业务对象，可将参数用作对象和流程中的操作的输入。

两种类型的参数均可以在 API 定义、通用参数和特定于操作的参数中进行配置。通用参数用在所有操作中，而在操作级别配置的参数仅可用于该操作。

要在 URL 路径、标头和请求正文中使用参数值，请将参数括在方括号中。例如：

http://staff.database.com/api/[version]/employee/[id]

当运行使用关联的 API 定义的流程并扩展参数时，URL 可能变为：

http://staff.database.com/api/v2/employee/290054

要使用方括号而不引用参数，文本必须以双方括号开头。例如，如果请求正文包含：

{

"favourite-colours":[["green", "red", "blue"]

}

调用 API 时，HTTP 请求会包含方括号：

{

"favourite-colours":["green", "red", "blue"]

}

在调用 API 时应用参数，并且如果已设置初始值，则从 API 定义中获取值。但是，如果已经公开了该参数，并且在流程操作中输入了作为输入的值，则将使用该值。
