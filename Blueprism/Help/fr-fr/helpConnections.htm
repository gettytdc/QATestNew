Boîte de dialogue des connexions

La boîte de dialogue des connexions vous permet d'enregistrer différentes configurations de connexion à la base de données. Toutes les connexions nécessitent un nom, ainsi que d'autres informations en fonction du type de connexion.

Serveur Blue Prism

La connexion à un serveur Blue Prism dédié est la configuration de connexion la plus sécurisée.

  • Nom du serveur – Vous devez fournir le nom d'hôte de la machine sur laquelle votre serveur Blue Prism fonctionne. La machine locale doit être en mesure de résoudre le nom d'hôte fourni.
  • Mode de connexion – Cela vous permet de sélectionner le mode de connexion utilisé pour les connexions entre le client et le serveur Blue Prism. Veuillez noter que ce réglage doit correspondre à la valeur correspondante définie dans la configuration du serveur Blue Prism.
  • Port – Vous devez fournir le port sur lequel votre serveur Blue Prism écoute.
  • Port de rappel – Spécifie un port sur lequel le client écoutera les rappels du serveur. Vous pouvez le laisser sur 0 pour permettre à un port disponible d'être choisi automatiquement. Notez que si vous utilisez cette option pour forcer un port spécifique, seule une instance d'une application Blue Prism pourra se connecter au serveur depuis cette machine. Ce réglage ne s'applique qu'à une connexion au serveur Blue Prism utilisant un mode de connexion .NET Remoting.

SQL Server (authentification SQL)

L'authentification SQL utilise les identifiants d'un utilisateur SQL pour accéder à la base de données.

Les bases de données autonomes sont prises en charge dans Blue Prism, mais elles doivent être créées manuellement à l'aide du script généré par l'option /getdbscript d'AutomateC.

  • Nom du serveur – Vous devez fournir le nom d'hôte de la machine sur laquelle votre serveur de base de données est hébergé. La machine locale doit être en mesure de résoudre le nom d'hôte fourni.
  • Nom de la base de données – Le nom de votre base de données Blue Prism tel qu'il apparaît sur le serveur de base de données. Si aucune base de données de ce type n'existe encore, vous pouvez choisir un nom de base de données ici. Vous pouvez ensuite créer une base de données via l'interface utilisateur Blue Prism comme décrit sur la page d'aide Créer une base de données.
  • ID utilisateur de la base de données – Indiquez ici le nom d'utilisateur de l'utilisateur de la base de données que Blue Prism doit utiliser pour accéder à la base de données.
  • Mot de passe de l'utilisateur de la base de données – Indiquez ici le mot de passe de l'utilisateur nommé dans le champ ci-dessus. Blue Prism ne révélera jamais ce mot de passe.
  • Paramètres de connexion SQL supplémentaires (serveur non Blue Prism uniquement) – Tout autre paramètre séparé par un point-virgule qui doit être joint à la chaîne de requête créée pour accéder à la base de données. par exemple.Encrypt=true;TrustServerCertificate=true.

Serveur SQL (authentification Windows)

L'authentification Windows utilise les identifiants d'utilisateur Windows actuellement connectés pour accéder à la base de données. L'accès doit être autorisé sur le serveur de base de données pour permettre à l'utilisateur Windows d'accéder à la base de données.

  • Nom du serveur – Vous devez fournir le nom d'hôte de la machine sur laquelle votre serveur de base de données est hébergé. La machine locale doit être en mesure de résoudre le nom d'hôte fourni.
  • Nom de la base de données – Le nom de votre base de données Blue Prism tel qu'il apparaît sur le serveur de base de données. Si aucune base de données de ce type n'existe encore, vous pouvez choisir un nom de base de données ici. Vous pouvez ensuite créer une base de données via l'interface utilisateur Blue Prism.

Groupe de disponibilité

Les deux options de groupe de disponibilité sont les mêmes que les options SQL Server correspondantes, mais se connectent à un groupe de disponibilité SQL Server. Le nom du serveur fait référence à l'écouteur du groupe de disponibilité, et deux réglages supplémentaires sont disponibles :

  • Port – Le port sur lequel l'écouteur du groupe de disponibilité écoute.
  • Basculement multi-sous-réseaux – Permet d'activer le basculement multi-sous-réseaux sur la connexion. Pour plus d'informations, reportez-vous à la documentation Microsoft à ce sujet.

Utilisation de plusieurs connexions

Vous souhaitez peut-être enregistrer plusieurs connexions, par exemple une base de données pour les tests et une autre pour la production. Cette boîte de dialogue vous permet de passer rapidement d'une connexion à l'autre. De nouvelles connexions sont ajoutées à l'aide du bouton « Nouvelle connexion ». Les informations relatives à la nouvelle connexion sont ensuite indiquées sur la droite.

Pour modifier une connexion existante, il suffit de la sélectionner dans la liste de gauche et de remplir les champs à droite. Quelle que soit la connexion sélectionnée, lorsque vous appuyez sur le bouton « OK », elle devient la connexion active utilisée par Blue Prism.

Pour des raisons de sécurité, seul le mot de passe de la connexion en cours est enregistré, vous devrez donc le saisir à nouveau lorsque vous passez d'une connexion à une autre.
