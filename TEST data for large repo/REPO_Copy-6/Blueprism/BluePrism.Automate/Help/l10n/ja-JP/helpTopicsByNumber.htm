トピック1064 - DateDiff関数の使用に対する警告

Blue Prismにはdatediff関数と同様の目的を達成するための優れた方法が用意されているため、datediff関数は可能な限り避けるべきです。特に、時刻の検出には使用しないでください。datediff関数で取得した結果は夏時間による変更の影響を受けます。時刻を検出するには、現在時刻の差を既知の時刻と比較して時間間隔を得ることをおすすめします。この時間間隔は固定値と比較する必要があります。

たとえば次の式 

DateDiff(6, Today(), Now()) < (18*60)

は、現在の時刻が午後6時より前かどうかを判断するためのものです。推奨される方法は以下のとおりです。 

LocalTime() - [MyReferenceTime] < [MyTimespan]

ここで、データアイテム「MyReferenceTime」は今晩の午後6時の瞬間を表し、データアイテム「MyTimespan」は値ゼロのタイムスパンを格納します。

トピック32788 - 認識されないアプリケーションタイプ

使用しているバージョンのアプリケーションマネージャーは、プロセスで指定されているアプリケーションタイプをサポートしていません。アプリケーション定義ウィザードを再実行して新しいアプリケーションタイプを選択するか、問題のアプリケーションタイプをサポートするBlue Prismからアプリケーションマネージャーのアップデート版を要求できます。

現在選択されているアプリケーションタイプを確認するには、プロセスをローカルのxmlファイルにエクスポートし、xml要素 "ApplicationTypeInfo"を探します。

トピック32798 - アプリケーションが接続されていません

ターゲットアプリケーションは、起動されるまで連携できません。連携を行う前に、アプリケーションが常に起動されるようにビジネスオブジェクトが構成されていることを確認してください。

トピック32808 - Java Runtime Environmentバージョン1.3.xに対する読み取り/書き込み/遷移の失敗

Blue PrismのJava AutomationはSunのJava Access Bridgeを使用して対象アプリケーションと連携します。Sunの製品には、Java Runtime Environment（JRE）のバージョン1.3のサポートが不完全でエラーが発生しやすいという既知の制限があります。「Java Access Bridgeのトラブルシューティング」ページも参照してください。

トピック32818 - 対象アプリケーション情報が設定されていません

対象アプリケーションと連携する前に、そのアプリケーションに関するいくつかの情報を登録する必要があります。 アプリケーションモデラーにアクセスしてこの情報を入力してから、ビジネスオブジェクトを保存してください。

ユーザー許可を参照してください。

トピック524 298 - ライセンスの制限のため、操作が拒否されました

システムマネージャーにインストールされている現在のライセンスでは、要求された操作が許可されていません。

最近ライセンスを変更した場合は、この変更を完全に有効にするために、（各マシンで）Blue Prismからログアウトして再度ログインする必要があります。

エラーがあると思われる場合は、技術的なサポートについて、Blue Prismまでお問い合わせください。
