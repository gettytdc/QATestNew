Java Access Bridge - 設定手順

概要

Java Access Bridge（JAB）のインストールには、自動と手動の2つの方法があります。前者が好ましい方法であり、ほとんどの場合これで十分です。後者については参考として説明します。

手動方式は、Java Runtime Environment（JRE）の非標準インストールでの使用が必要な場合があります。 このようなインストールは、ベンダーがソフトウェアを実行する環境のコントロールを維持する必要があるため、またはベンダーが特別な目的のためにJREを変更するために使用します。

前提条件

Java Access Bridge（JAB）が使用されるJava Runtime Environment（JRE）を最初にインストールする必要があります。

要件

本記事の執筆時点で、Java Access Bridge（JAB）の最新バージョンは2.0.1です。 ただし、このバージョンには重大な欠陥があり、Blue Prismでの使用には適していません。JREバージョン1.2.2以降で使用する場合、JABバージョン2.0.0をお勧めします。JAB 2.0.0は、リクエストに応じてBlue Prismから入手できます。JREバージョンが1.2.2より前のものである場合は、Blue Prismにご相談ください。

自動インストール

自動インストールを開始するには、インストール実行ファイルを実行して画面の指示に従います。完了したら、PCを再起動します。

手動インストール

手動インストールは、非標準のJREにのみ使用します。必要に応じて、自動インストーラーが標準のJREを見つけて修正します。

1.ファイルを抽出する

インストールアーカイブaccessbridge-2_0_0-manual_install.zipの内容を一時ディレクトリに抽出します。

2.システムファイルをコピーする

JavaAccessBridge.dll、JAWTAccessBridge.dll、WindowsAccessBridge.dllの各ファイルを、次の場所にあるWindowsシステムディレクトリにコピーします。 

 c:\windows\system32\

自動インストーラーを実行した場合、これらのファイルがすでに存在している可能性がありますのでご注意ください。 この場合はファイルを置き換える必要はありません（それらのファイルがバージョン1.0などJava Access Bridgeの以前のバージョンに対応している場合を除く）。

3.ターゲットJREを見つける

最初にJREインストールを見つけます。インストールするJava Access Bridge（JAB）を一度に複数のJREインストールで使用できます。Sun Microsystemsの公式Java Webブラウザープラグインなどの製品では別の新しいJREがインストールされるため、実際にこの方法が必要かつ望ましい場合もあります。Webページに組み込まれたJavaアプリケーションを自動化できるようにするには、JABをこのJREにインストールする必要があります（その他の任意の選択と同様）。複数のJREにインストールするには、各JREインストールディレクトリに対して以下の手順を繰り返します。

JREの標準インストールは、通常、次のようなディレクトリにあります。 

 c:\Program Files\Java\jdk1.5.0\jre\
 c:\Program Files\Java\jre1.5.0_05

JREの非標準インストールは、通常、対象アプリケーションがインストールされたディレクトリのサブディレクトリにあります。次に例を示します。 

 c:\Program Files\MySpecialApplication\JRE\

4.拡張ディレクトリを見つける

インストールディレクトリの下のlib\extフォルダーに移動します。次に例を示します。 

 c:\Program Files\Java\jdk1.5.0\jre\lib\ext
 c:\Program Files\Java\jre1.5.0_05\lib\ext

5.Javaファイルをコピーする

access-bridge.jarという名前のファイルを、前のステップで識別された「ext」ディレクトリにコピーします。

ターゲットJREインストールのバージョン（1.Xの形式の値）を識別してください。 「ext」ディレクトリにjaccess-1_X.jarという名前のファイル（XはJREバージョンの一部として識別される番号）をコピーして、名前をjaccess.jarに変更します。

一致するファイルがない場合は、代わりに、利用可能なファイルのうち次に大きいバージョン番号のファイルを使用します（たとえば、1.5または1.6で、一致するファイルが存在しない場合は、ファイルjaccess-1.4.jarを使用してください）。

6.プロパティファイルを作成する

「lib」フォルダー（「ext」フォルダーの1つ上のレベル）にaccessibility.propertiesという名前のファイルが存在するかどうか確認し、必要に応じてテキストエディターを使用してこれを作成します。 名前を正しくつづってください。ファイルに以下の行が含まれていない場合は、ファイルの末尾で改行して、この行を追加します。 

assistive_technologies=com.sun.java.accessibility.AccessBridge

この行しかファイルに含まれていなくてもまったくかまいません。

7.クリーンアップ

インストールアーカイブ（accessbridge-2_0_0-manual_install.zip）の内容が抽出された一時ディレクトリを削除します。

PCを再起動します。

ノート

対象となるJREをアップグレードまたは変更（たとえばJava Runtime Environmentの作成元であるSun Microsystemsが発行したセキュリティ更新として）するたびに、Java Access Bridgeのインストールを繰り返し、新しいJREが正しい拡張子を持つのを確認し、Access Bridgeと連携させる必要があります。

新しいJREを使用中のマシンにインストールするたびに（たとえば新しいソフトウェアと併用するため）、既存のJREに対しても、Java Access Bridgeのインストールを繰り返す必要がある場合があります。
