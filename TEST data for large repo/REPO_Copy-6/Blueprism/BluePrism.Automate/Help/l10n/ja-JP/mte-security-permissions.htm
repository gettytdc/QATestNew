セキュリティの役割の許可

Multi-Team Environments機能の一部として、既存の許可にさまざまな変更が加えられています。追加された許可もあれば、更新または再割り当てされた許可もあります。6.3より前のバージョンからアップグレードした場合、既存のすべてのユーザーは、現在役割から与えられているアクセス権を維持します。

Multi-Team Environmentsで粒度が向上したため、すべてのユーザーにそのユーザーが必要とする許可のみが与えられていることを確認するために、セキュリティロール許可を見直すことが推奨されます。

新しい許可

エリア

許可

説明

オブジェクトスタジオ

ビジネスオブジェクトを実行

ユーザーはオブジェクトを実行できますが、オブジェクトを開いたり、編集したりすることはできません。

ビジネスオブジェクトアクセス権を管理

ユーザーはオブジェクトグループに対するアクセス権をコントロールできます。

Webサービスとしてビジネスオブジェクトを実行

ユーザーはWebサービスとして公開されているオブジェクトを呼び出すことができます。

プロセススタジオ

プロセスを実行

ユーザーはプロセスを実行できますが、プロセスを開いたり、編集したりすることはできません。

Webサービスとしてプロセスを実行

ユーザーはWebサービスとして公開されているプロセスを呼び出すことができます。

コントロールルーム

セッション管理へのフルアクセス

コントロールリソース許可からも同じアクセス権が与えられます。

セッション管理への読み取りアクセス
リソースを表示許可からも同じアクセス権が与えられます。

オブジェクトスタジオ

ビジネスオブジェクトを比較

オブジェクト定義を表示許可でユーザーはビジネスオブジェクトを比較できます。

ビジネスオブジェクトを作成

ビジネスオブジェクトを作成/複製の名前が変更され、オブジェクトを複製する許可が追加されました。

ビジネスオブジェクトをテスト

ビジネスオブジェクトの実行許可からも同じアクセス権が与えられます。

ビジネスオブジェクトを表示

ビジネスオブジェクト定義を表示の名前が変更されました。

プロセススタジオ

プロセスを比較

プロセス定義を表示許可でユーザーはプロセスを比較できます。

プロセスを作成

プロセスを作成/複製の名前が変更され、プロセスを複製する許可が追加されました。

プロセスをテスト

プロセスを実行許可からも同じアクセス権が与えられます。

プロセスを表示

プロセス定義を表示の名前が変更されました。

リソースに対する新しい許可も導入されました。詳細情報については、「リソースのアクセス権」を参照してください。
