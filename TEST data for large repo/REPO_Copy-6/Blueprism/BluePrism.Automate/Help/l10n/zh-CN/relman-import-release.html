导入发行版向导

概述

导入发行版向导支持将发行版、技能文件和遗留流程/对象导入当前环境。

具有“发行版管理器—导入发行版”权限的任何人都可以导入兼容文件，随后他们需要具有相应的权限才能导入或修改相关组件。例如，如果发行版包含流程、凭据和工作队列，则运行该向导的用户必须具有以下权限才能成功导入该发行版：

·         导入发行版

·         导入流程

·         安全—管理凭据

·         工作流程—工作队列配置

这些权限是在不使用发行版文件的情况下直接在产品中创建和/或修改单个组件所需的权限。

导入文件

以下过程描述了如何导入文件并处理出现的任何冲突。

1.       在 Blue Prism 客户端中，选择“文件”>“导入”。

还可以从“发行版”选项卡导入发行版。

此时会打开“导入发行版”向导并显示“选择输入文件”页面。

2.       点击“浏览”并找到所需文件。

所选文件必须是以下类型之一：

o  发行版管理器创建的 Blue Prism 发行版文件 (.bprelease)

o   Blue Prism 技能文件 (.bpskill)

o   流程或对象，导出为 .xml 文件

3.       点击“下一步”。

系统将分析所选文件以验证发行版的内容。如果发现发行版没有任何问题，则导入成功完成。

如果文件中的组件与当前环境中的相应组件之间存在任何冲突，则会显示向导的解决导入冲突页面。

任何导入相关问题都与相应的解决方案一起列出。

 

4.       在“解决方案”列中，为每个组件选择所需的操作。在需要进一步配置的情况下，系统将显示一个对话框，用于输入新数据。

只有在解决了所有冲突并且为需要配置的组件输入了有效数据后，才能进行导入。

有许多规则可确定导入文件的可用选项。有关所有规则和选项的列表，请参阅[发行版管理器—导入帮助主题链接]。

5.       点击“完成”将发行版的组件保存到数据库并完成导入。

命令行导入

此外，还可通过命令行导入发行版、技能、对象和流程。有关更多信息，请参阅[命令行选项主题链接]。
