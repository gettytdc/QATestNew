将流程或业务对象公开为 Web 服务

本演练介绍在将流程或业务对象作为 Web 服务提供时所需的步骤。 在公开后，系统还会提供有关如何验证 Web 服务是否已发布以及如何访问 WSDL 的信息。

  1. 进入系统管理器，然后选择“流程”或“业务对象”区域。
  2. 选择“流程公开”或“业务对象公开”选项卡，然后点击“公开”链接以启动向导。
  3. 此时将显示一个列表，其中列有当前还未公开为 Web 服务的流程或业务对象。选择要公开的流程或业务对象，然后点击“下一步”。
  4. 向导将根据流程或业务对象的名称来推荐适用的 Web 服务名称。 可在此时更改名称，但如果名称与先前已公开的 Blue Prism Web 服务重复，或者检测到任何无效字符，则会显示警告。点击“自动更正”可删除任何无效字符，并确保名称唯一。
  5. 在名称可以接受后，点击“完成”。
  6. 新公开的 Web 服务名称现将显示在流程或业务对象名称旁边的列表中。

可以使用以下 URL（还提供了 WSDL 的地址）来确认从 Blue Prism 中公开的 Web 服务。

http://[machinename]:[port]/ws/

在本示例中，计算机名称是 resource001，而且使用了默认端口 8181：

http://resource001:8181/ws/

其中，运行时资源被配置为应用基于证书的加密（URL 需要加上前缀 HTTPS）：

https://resource001:8181/ws/
