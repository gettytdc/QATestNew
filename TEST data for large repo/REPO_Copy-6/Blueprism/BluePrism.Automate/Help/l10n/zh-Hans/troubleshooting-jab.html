Java Access Bridge

简介

此页面提供使用 Blue Prism 的 Java Access Bridge 功能的一些常见问题和建议的解决方法

突出显示或访问元素时，目标应用程序崩溃

如果元素探测正确，但目标应用程序在尝试突出显示这或与这些元素进行交互时崩溃，则该问题可能是由 Java Access Bridge 组件中的缺陷导致的，可能会是在单个屏幕上存在大量元素（大约 200 个或更多）时出现问题。您可以通过在应用程序建模器中启用“descendtree”目标应用程序选项来防止崩溃。此模式的速度明显较慢，因此请仅在必要时使用。

JAB 版本

建议对所安装的 JRE 使用最新的 JAB 版本。从 JRE7u6 开始，JAB 2.0.3 与 java 安装程序捆绑在一起。从 Jre7u60 开始，捆绑 JAB 2.0.4。

JRE 版本JAB 版本
JRE5 以下版本2.0.1 或 2.0.0
JRE5 及更高版本2.0.2
JRE6 及更高版本2.0.2
JRE7 及更高版本2.0.4

Blueprism 可以提供 BPJabInstaller-1.0.2.exe，它可以安装没有捆绑 JAB 的各种 JRE 版本。

当 JAB 与 java 安装程序捆绑在一起时，必须启用它，这可以使用以下命令行完成。打开命令提示符并切换到 java bin 目录，例如 C:\Program Files\Java\Jre7\bin，然后运行以下命令：

jabswitch.exe /enable

也可以使用 Windows“轻松访问”控制面板启用 JAB（可以通过按 Windows + U 激活）。“轻松访问”控制面板具有 Java Access Bridge 复选框。

探测工具不能识别元素

如果在将鼠标悬停在元素上时探测工具无法对其进行识别，请首先检查以下常见原因：a) 安装 Java Access Bridge 后无法重新启动，b) 使用错误的 Java Access Bridge 版本

尝试写入到文本框时给出错误消息

错误消息“Call to setTextContents failed. This method is known not to work against version 1.3.x of the Java Runtime Environment”（调用 setTextContents 失败。已知此方法对于 1.3.x 版本的 Java Runtime Environment 无法正常实施）表示您正在尝试在 JRE 1.3 下运行的应用程序中写入 Java 文本框。遗憾的是，由于与 JRE 版本对应的 Java Access Bridge 连接器中缺少实施，因此可能无法进行此操作。

在所有情况下都有变通方法，但用户方面可能需要有一些自己的想法和创造性。下方是针对常见问题提出的建议变通方法列表：

常规变通方法—升级 Java Runtime Environment (JRE)
大多数情况下，目标应用程序完全会像升级之前一样正常运行，但您仍应在执行升级前考虑任何潜在问题的相关风险。Blue Prism 对于版本 1.4.0 或更高版本能够可靠地运行。
应对写入问题的变通方法
如果在您向元素写入数据时遇到问题，则您可以向应用程序发送击键作为替代方法。其中涉及激活包含所关注元素的窗口，聚焦准备好接收键盘输入的元素，以及使用导航环节发送击键。此等击键可能需要包含退格键或删除键以清除元素中的任何现有数据。
