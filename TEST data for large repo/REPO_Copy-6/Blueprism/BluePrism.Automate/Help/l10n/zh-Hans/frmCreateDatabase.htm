创建一个新的数据库

1. 设置连接信息

首先访问数据库连接窗体，输入要在其上创建数据库的服务器的详细信息，并输入新数据库应使用的名称。

2. 确保数据库未被使用

Blue Prism 将使用连接窗体中提供的名称创建一个数据库（或者，如果已存在共用该名称的现有数据库，则修改该数据库）。如果具有该名称的数据库已经存在，则必须确保其他应用程序未使用该数据库。这包括在其他资源 PC 上运行的 Blue Prism 的其他实例。继续之前，确保当前没有其他应用程序正在使用该数据库。

3. 选择是否清除任何现有数据库

如果已存在具有指定名称的数据库，则可以修改现有实例，也可以清除（即永久和彻底删除）现有数据库，并在其位置创建一个同名的新数据库。建议选择后一种做法。

如果用于运行 Blue Prism 的数据库用户名没有 DROP 和 CREATE 数据库的权限，则前一种选择可能很有用。如果不存在具有指定名称的数据库，则选择“清除”做法将不起作用。

4. 选择身份验证方法

可以选择标准的“Blue Prism 身份验证”或者单点登录身份验证。“单点登录”通常仅由具备良好管理的网络的大型组织使用。此选项将需要网络管理员的协助来配置 Microsoft Active Directory 选项。

如果不确定，则选择默认选项“Blue Prism 身份验证”。

5. 创建数据库

从 Blue Prism 的主菜单中，选择“创建数据库”选项。必须注销 Blue Prism 才能这样做。在弹出的窗体上，确认“连接”对话框中给出的密码，点击“确定”并等待。创建数据库后，您将收到一条确认消息。从此时开始，您可以登录到新数据库，如下所述。

6. 登录

如果您成功配置了“单点登录”，则应会自动登录。如果遇到问题，检查您的设置（包括 Active Directory 组成员身份等），然后重试。请注意，在注销并再次登录之前，Active Directory 更改可能不会生效。

如果您使用的是“Blue Prism 身份验证”，则可以使用用户名“admin”和密码“admin”登录到新数据库。强烈建议您更改此管理员帐户的密码，以防止不必要的访问。

登录后的首要任务之一可能是：

  • 为系统的其他用户创建用户帐户。
  • 为新创建的用户管理用户角色和权限。如果您使用的是“单点登录”，这尤其重要。
