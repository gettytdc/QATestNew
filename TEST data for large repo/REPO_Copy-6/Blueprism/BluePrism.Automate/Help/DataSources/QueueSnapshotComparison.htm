Blue Prism

Queue Snapshot Comparison

BPDS_QueueSnapshotComparison

Returns queue snapshot data for today - in the timezone specified in the snapshot configuration - and compares it against the same statistic from a previous day's snapshot data.

Source: Blue Prism Database - Live tables and MI Repository (requires collect utilization and productivity statistics to be enabled under Settings - Reporting).

Sample Chart

Parameters

Parameter Name Data Type Description Validation
@QueueName Text The name of the queue to be queried. None.
@NumberOfSnapshottedDaysPrevious Integer

Determines which day's snapshots will be used when comparing data with the snapshots for the current day.
The query will count back though the previous snapshotted days by the number specified.
For example, if today is Monday and snapshots are scheduled to occur every day, entering a value of 1 will compare the snapshots for today (Monday) and yesterday (Sunday). A value of 2 will compare today's snapshot data with that taken on Saturday, and so on.
By contrast if snapshots are scheduled to only occur Monday-Friday then entering a value of 1 will compare the snapshots for today (Monday) and the previous Friday. A value of 2 will compare today's snapshot data with that taken on Thursday, and so on.

 None.
@ColumnIdentifier Integer The specific metric from the snapshot data that will be used to compare the two days - if a value is not specified, the default setting, Total Items, is used.

Possible values are:

  • 1 - Total items: the total number of items in the queue at the time of the snapshot
  • 2 - Items pending: the total number of pending items in the queue at the time of the snapshot
  • 3 - Items completed: the total number of completed items in the queue at the time of the snapshot
  • 4 - Items referred: the total number of referred items in the queue at the time of the snapshot
  • 5 - New items delta: the total number of new items brought in to the queue since the last snapshot was taken
  • 6 - Completed items delta: the total number of items marked as completed in the queue since the last snapshot was taken
  • 7 - Referred items delta: the total number of items marked as referred in the queue since the last snapshot was taken
  • 8 - Total worktime completed: the total work time (in seconds) of all items completed in the queue since the last snapshot
  • 9 - Total worktime referred: the total work time (in seconds) of all items referred in the queue since the last snapshot
  • 10 - Total idle time: the total time spent waiting in the queue (in seconds) of all items completed or referred since the last snapshot
  • 11 - Total new since midnight: the total number of new items brought into the queue since midnight (in the time zone configured for the queue)
  • 12 - Total new in the last 24 hours: the total number of new items brought into the queue in the last 24 hours
  • 13 - Average completed work time: the average work time (in seconds) of an item marked completed in the queue this snapshot
  • 14 - Average referred work time: the average work time (in seconds) of an item marked referred in the queue this snapshot
  • 15 - Average idle time: the average time spent waiting in the queue(in seconds) of an item completed or referred this snapshot
 Values 1 - 15 can be entered.
@TimeRangeStartTime Text The start time of the time range for the displayed data.
  • This parameter is optional.
  • If not specified then all time intervals will be displayed for the configuration applied to the queue.
  • Required format: {HH:mm} (e.g. '01:15', '13:50', '22:10', '07:00')
@TimeRangeEndTime Text The end time of the time range for the displayed data.
  • This parameter is optional.
  • If not specified then all time intervals will be displayed for the configuration applied to the queue.
  • Required format: {HH:mm} (e.g. '01:15', '13:50', '22:10', '07:00')
