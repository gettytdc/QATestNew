Blue Prism Login

Overview

Blue Prism requires a user to provide authentication before they are able to perform any tasks within the environment defined in the currently configured connection.

There are two methods of authentication supported by Blue Prism, namely 'Blue Prism user authentication' and 'Active Directory authentication' (or 'single sign on'). The type of authentication to be used is set when the Blue Prism database is created and cannot be changed on existing databases.

This login screen provides the interface with which a user can authenticate to the product and enter Blue Prism.

Blue Prism authentication

This requires each user to have a corresponding user configured in System Manager. In order to enter the product, the user must enter their user name and password and click the Sign In button.

For further details on maintaining users within a Blue Prism environment see the System Manager / Users help page.

Active Directory authentication

This requires a user to be registered within a pre-specified Active Directory group in order to log into Blue Prism.

See the Single Sign On help page for further information.

In order to authenticate and log into Blue Prism, the user must click the Sign In button.

Choosing a default connection.

The connections defined on this machine are available in the Connections dropdown list, the current connection being the one selected.

The current connection can be changed by selecting another from the dropdown list. An attempt is made to connect using the specified details - if the attempt fails, an error message is displayed and the login controls are disabled. Otherwise, the user may continue to log into the newly chosen connection.

Blue Prism will automatically connect using the user's last chosen connection.

Further login screen options

Further options are available within System Manager for the environment-specific presentation of the login screen.

A list of the users registered within the current connection can be displayed (for Blue Prism authenticated connections only).

The user name can be pre-populated from the last time that the user logged into the selected connection.

These options are available within System Manager - see the Logon Options help page for more information.
