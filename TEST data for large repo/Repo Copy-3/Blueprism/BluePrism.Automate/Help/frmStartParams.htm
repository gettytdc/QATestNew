Startup Parameters

The Startup Parameters form allows you to set the startup parameters for one or more pending processes. You can access the form by right-clicking the pending processes in Control Room and selecting Startup Parameters. Alternatively, when you start the selected processes, the Startup Parameters form will be displayed if one or more of the processes selected contains startup parameters that have not been set.

The form displays the sessions, grouped by process, in the treeview on the left with the input parameters displayed on the right.

Parameters of type collection cannot be set from this dialog - these parameters are only valid when the process is to be called as a subprocess.

To set a parameter on a single session.

On the Startup Parameters form click the session you wish to modify. Modify the parameters to the right of the treeview. When you modify a parameter, a tick next to the selected session will indicate that some parameters have been set on the session.

To set a parameter on multiple sessions.

On the Startup Parameters form click the process which groups all the sessions together and then modify the parameters to the right of the treeview. When an input parameter has been set, a tick will appear on the right-hand side of the parameter input box. A similar tick will also appear in the left-hand treeview against the session and the process. You can click these ticks to clear the input parameters. Clicking the tick against the session will clear all parameters within the session and clicking the tick against the process will clear all parameters for all sessions of that process.

Once you have set the startup parameters, you can either start the process from the dialog box or save the startup parameters. If you save the parameters, the processes can be started from control room without the Startup Parameters dialog appearing.

If you fail to set a startup parameter you will be warned when either starting the process or saving the parameters. If you run the process without the startup parameters being set, the default values for the data items will be used.
