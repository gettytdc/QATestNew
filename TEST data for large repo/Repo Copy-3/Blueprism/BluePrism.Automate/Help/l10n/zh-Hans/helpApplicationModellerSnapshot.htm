应用程序建模器—诊断快照

概述

“诊断快照”功能可用于对以下问题进行故障排除：模型中没有预期会适时出现在特定实例中的元素。当业务对象在运行时，如果 Blue Prism 无法找到特定元素，诊断快照可以完全捕获 Blue Prism 能够识别的每个 UI 特性的所有细节。这有助于在找不到元素时构建应用程序的状态图。

使用

“诊断快照”功能适用于浏览器、Windows 和 Java 应用程序类型。要使用诊断快照，您必须首先针对业务对象打开此功能。在“应用程序建模器”窗口中选择应用程序模型中的根节点，并按下“诊断 ...”按钮。点击该按钮后，将出现一个对话框，以便您选择一个可用的快照方法。对于 Windows 和 Java 应用程序类型，只有一个标记为“快照”的选项可供使用；对于浏览器应用程序类型，可供使用的选项有“HTML 快照”和“源代码捕获”。请确保至少选中一个复选框，然后点击“确定”。

启用此功能后，您可以右键点击与 UI 功能对应的应用程序模型中的任意元素。右键点击此元素后，系统将显示一个上下文菜单，其中会有“创建错误快照”菜单项。选择此菜单项后将选中或取消选中菜单项旁的复选框。

在遍历对象时，如果应用程序管理器无法识别任何选中的元素，那么它现在将获取整个模型的快照，并将快照文本转储到日志中。对于浏览器应用程序类型，如果选择了该选项，它还将转储 HTML 的源代码抓图。

输出

您可在系统管理器中打开业务对象日志，以查看快照的输出。

快照文本的格式使用制表符缩进表示元素的树结构，后面跟着一行属性值对。

HTML 快照的格式是嵌入到 XML 文档的 HTML 块。HTML 快照文本会通过电子邮件发送给 Blue Prism 开发团队；然后，开发团队可以借助 BPDiagnostics 中随附的工具使用这个捕获文件来重新创建客户 Web 页面，以开展非现场集成测试。

立即开始

这个功能允许您根据当前应用程序状态立即创建快照文本或源代码捕获文本。在“应用程序建模器”窗口中选择应用程序模型中的根节点，并按下“诊断 ...”按钮。点击该按钮后，将出现一个对话框，以便您选择一个可用的立即开始按钮。按下所选按钮后，即会针对应用程序的现有状态发起快照或源代码捕获。快照文本将通过弹出式对话框的方式呈现。
