业务对象—初始化和清理

业务对象包含两个强制操作，即初始化和清理。这两个操作无法删除或重命名，其他操作也不能与它们同名。

您不太可能需要使用初始化和清理页面，因为大多数情况下都无需使用

初始化

在创建业务对象时，系统会自动调用初始化页面。如果要在使用业务对象之前先执行维护操作，上述做法可能会很有帮助。

页面必须有效（也就是说，所有其他页面都必须有效），初始化页面才能运行。尤其是，以开始环节为起点的每一条可能的流程路径中都必须有一个结束环节。如果初始化页面无法运行，业务对象就不会被加载，而且使用该对象的流程将会失败。

清理

当业务对象的父流程结束时，系统会自动调用清理页面。流程会在运行完成时、在出现错误和故障后或在用户从控制室中将其停止时结束。

它主要用于使与业务对象拥有相同生命周期的资源关闭运行。它可以保证某些操作在父流程停止后仍能得以运行，无论父流程有没有出现错误。

页面必须有效（也就是说，所有其他页面都必须有效），初始化页面才能运行。尤其是，以开始环节为起点的每一条可能的流程路径中都必须有一个结束环节。
