连接配置

概述

可从登录页访问的“连接配置”窗体提供可将多个连接配置为 Blue Prism 数据库和服务器的界面。

软件可以使用 SQL Server 或 Windows 身份验证连接到 SQL Server 2005 及更高版本。或者，软件可以通过运行在可用网络中的服务器上的 Blue Prism 服务器进行连接。

默认情况下，按照每台计算机来配置连接，即连接列表可供当前计算机上的所有用户使用，并且任何用户做出的任何更改都将对下一个用户可见，以便在同一台计算机上使用 Blue Prism。通过使用 /useropts 开关来启动 Blue Prism，可将默认的做法改变为按照每个用户来创建连接列表。

配置现有连接

可以通过在窗体左侧的列表中选择连接，然后在右侧显示的字段中编辑详细信息来配置连接。

连接名称

此名称用于在产品中的登录屏幕或 BP 服务器配置中描述连接，以指示 Blue Prism 应使用的连接。

连接类型

Blue Prism 中支持多种连接（和身份验证）类型：—

SQL Server（SQL 身份验证）
直接连接到 SQL Server 实例，使用 SQL 登录凭据（即在数据库上设置的用户名和密码）进行身份验证。
SQL Server（Windows 身份验证）
直接连接到 SQL Server 实例，通过 Windows 用户帐户进行身份验证。
Blue Prism 服务器
连接到现有的 Blue Prism 服务器—服务器维护与数据库本身的连接。
可用性组（SQL 身份验证）
直接连接到 SQL Server 可用性组，使用 SQL 登录凭据（即在数据库上设置的用户名和密码）进行身份验证。
可用性组（Windows 身份验证）
直接连接到 SQL Server 可用性组，通过 Windows 用户帐户进行身份验证。

有关连接类型之间的差异的更详尽讨论，请参阅 Blue Prism 服务器。

数据库服务器（仅限 SQL Server）

Blue Prism 应该连接的数据库实例—例如 localhost\SQLExpress 将连接到在本地计算机上运行的名为 SQLExpress 的实例。

数据库名称（仅限 SQL Server）

Blue Prism 将其数据存储到的指定实例上的数据库名称。

用户 ID（仅限 SQL Server—SQL 身份验证）

Blue Prism 应用于向 SQL Server 进行身份验证的 SQL 用户的 ID

密码（仅限 SQL Server—SQL 身份验证）

Blue Prism 应用于向 SQL Server 进行身份验证的密码。

其他 SQL 连接参数（仅限非 BP 服务器）

应该附加到为访问数据库而创建的查询字符串中的任何其他参数（用分号分隔），例如Encrypt=true;TrustServerCertificate=true。

多子网故障转移

允许在连接上启用多子网故障转移。有关详细信息，请参阅 Microsoft 的文档。

Blue Prism 服务器（仅限 BP 服务器）

客户端应连接的 Blue Prism 服务器的网络地址。

连接模式（仅限 BP 服务器）

以下连接模式可用于 Blue Prism 客户端和 Blue Prism 服务器之间的连接。请注意，此设置必须与 Blue Prism 服务器配置中设置的相应值匹配。

WCF：采用消息加密和 Windows 身份验证的 SOAP
  • 需要设备之间的信任关系：是
  • Blue Prism 身份验证模式：Blue Prism 本机/单点登录
  • 需要服务器端证书：否
  • 传输：SOAP over HTTP

仅加密消息内容。SOAP 和 HTTP 标头保持未加密状态，可为复杂的路由、负载均衡器、代理等提供帮助。客户端和服务器标识通过 Windows/Active Directory 进行验证。

WCF：采用传输加密和 Windows 身份验证的 SOAP
  • 需要设备之间的信任关系：是
  • Blue Prism 身份验证模式：Blue Prism 本机/单点登录
  • 需要服务器端证书：是
  • 传输：SOAP over HTTPS

包含 SOAP 标头的传输使用基于证书的加密方式进行加密。客户端和服务器标识通过 Windows/Active Directory 进行验证。

WCF：采用传输加密的 SOAP
  • 需要设备之间的信任关系：否
  • Blue Prism 身份验证模式：Blue Prism 原生
  • 需要服务器端证书：是
  • 传输：SOAP over HTTPS

包含 SOAP 标头的传输使用基于证书的加密方式进行加密。使用证书验证服务器标识。

.NET Remoting：安全
  • 需要设备之间的信任关系：是
  • Blue Prism 身份验证模式：Blue Prism 本机/单点登录
  • 需要服务器端证书：是
  • 传输：TcpChannel over SChannel

为向后兼容性而提供。在客户端和服务器之间协商加密。客户端和服务器标识通过 Windows/Active Directory 进行验证。

.NET Remoting：不安全
  • 需要设备之间的信任关系：否
  • Blue Prism 身份验证模式：Blue Prism 原生
  • 需要服务器端证书：否
  • 传输：TcpChannel

不推荐—为向后兼容性而提供。连接安全性需要完全由第三方解决方案提供。

WCF：不安全
  • 需要设备之间的信任关系：是
  • Blue Prism 身份验证模式：Blue Prism 本机/单点登录
  • 需要服务器端证书：是
  • 传输：SOAP over HTTP

不推荐。连接安全性需要完全由第三方解决方案提供。

服务器端口（仅限 BP 服务器）

指定的 Blue Prism 服务器在其上侦听连接的端口。

回调端口（仅带有 .NET Remoting 的 BP 服务器）

当使用 .NET Remoting 连接模式连接到 Blue Prism 服务器时，客户端允许它向其“回调”。允许双向通信意味着某些服务器操作可以更及时地提供进度反馈。将此值保留为默认值（零）意味着在建立与服务器的连接时将分配任意回调端口。如果必须使用特定端口（例如，为了打开防火墙上的端口），则应在此处设置该端口。

请注意，如果配置了特定端口，则此计算机上只有一个 Blue Prism 实例可以在任何时候使用此连接。由于指定的回调端口正在使用中，从当前计算机连接到服务器的任何进一步尝试都将失败。

测试连接

点击“测试连接”按钮，可以测试当前显示在右侧“当前连接”详细信息窗格中的连接。

将尝试连接到已配置的 Blue Prism 数据库，并且系统将显示一个包含该尝试结果的消息框。

创建新连接

点击窗体左下角的“新连接”按钮，即可创建新连接。

系统将使用空白详细信息创建任意命名的连接，并将该连接显示在详细信息窗格中。

删除连接

通过在选择窗格中选择连接并按“删除连接”按钮，可以删除连接。

注意： 此操作没有确认提示。

保存更改

正确配置连接后，按“确定”按钮即可保存该连接。

此操作将为此计算机上的所有用户保存连接配置（除非使用 /useropts 命令行开关启动 Blue Prism），并将当前选定的连接设置为 Blue Prism 客户端中的当前连接。

放弃更改

如果有任何错误，可以通过按“取消”按钮来放弃更改—此时窗体会关闭，而不保存任何更改。

Windows Vista/Windows 7 上的用户帐户控制

请注意，在 Windows Vista 及更高版本上保存连接配置时，可能需要提高权限。有关更多信息，请参阅用户帐户控制帮助页面。
