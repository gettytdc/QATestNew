浏览器集成

简介

此页面提供在 Blue Prism 中使用浏览器集成的一些常见问题和建议的解决方法

多个浏览器会话无法正确附加

Internet Explorer 版本 7+ 引入了一项功能，这意味着浏览器中的单独会话可以在衍生自主 IE 进程的单独进程中运行。

这会影响以受保护模式在 Windows Vista 或 Windows 7 中运行的 IE7，以及所有 Windows 版本中的 IE8 +。

子索引

Blueprism 版本 4.1.3 及更高版本包含这样一个功能，允许您指定一个要向其附加的进程“子索引”。根据会话进程的创建顺序，每个会话会分配有一个唯一索引。例如，主父进程的索引为 0，则进程的首个选项卡的子索引为 1。子索引参数在导航环节的附加操作中指定。

TabProcGrowth

如果以上方法对您不适用，您可以改为将 Internet Explorer 配置为不创建多个进程。此操作可通过更改注册设置来完成，具体为更改注册表项： HKEY_CURRENT_USER\Software\Microsoft\Internet Explorer\Main ，应将名为 TabProcGrowth 的 DWORD 值设置为以下值之一：—

0（零）
禁用 IE 安全区域的保护模式，让 Internet Explorer 在同一进程中打开选项卡和框架。
1（一）

启用保护模式，确保给定框架中的所有选项卡均在一个进程中运行。

注意，此种设置下，需要使用进程流来启动并单独附加到所需的 IE 实例。仍将创建两个进程，一个用于框架，另一个用于此框架中的任一选项卡。

更改该值需要重启 Internet Explorer，然后 Internet Explorer 才能获取新值。

未对齐突出显示

如果您发现探测或突出显示探测元素会导致突出显示框与您正在探测或突出显示的元素未对齐，请检查 Internet Explorer 中的缩放设置（以及“控制面板”中的显示）。
