工作流—工作队列

概述

工作队列从系统管理器的“工作流程”部分进行管理。这包括创建、删除、重命名和配置队列。

管理队列

队列列表显示左侧列表中。所选队列的详细信息显示在右侧。

创建新队列
点击“新建”按钮，然后使用右侧的面板为您的队列命名并配置其余详细信息。
删除队列
在左侧列表中选择感兴趣的队列，然后使用“删除”按钮。只能删除空队列。您应首先确保队列中的所有项都已处理（或手动标记为异常），然后从队列中删除。
重命名队列
在左侧列表中选择感兴趣的队列，然后在右侧编辑名称。点击“应用”以完成更改。
更改队列的状态
您可以使用右侧显示的链接暂停队列。暂停的队列对于尝试从中选择案例的流程显示为空。之后可以随时恢复暂停的队列。
确保在执行此操作之前已在左侧选择了正确的队列。
配置键字段
使用 Process Studio 集合将项输入队列。在此集合中的字段中，其中一个可用作每个项的键。该键用于在从控制室监视队列活动时标识项。例如，如果您正在更新列表中的银行账户，那么您可能希望使用账号作为键字段。
如果不希望这样，您不必设置键字段。 例如，这可能适用于正在处理的信息是保密的情况。
设置最大尝试次数
可以选择要处理、部分更新和未完成的案例。然后，将在后面的环节再次从队列中选择它以供处理。您在此处输入的值会限制允许发生的次数。达到此限制后，案例将自动标记为异常。
要更改“最大尝试次数”设置，请在左侧列表中选择感兴趣的队列，然后在右侧编辑值。点击“应用”以完成更改。
设置加密器/解密器
每个工作队列项所具有的数据默认情况下以纯文本形式存储在数据库中。队列可配置为在将数据保存到队列时自动加密数据，并在从队列中检索数据时自动解密数据。
要在队列中启用加密功能，必须为当前环境配置加密方案。
要更改加密方案或禁用加密，请在窗口左侧选择队列，然后使用复选框和下拉列表选择队列的加密状态。
如果无法在服务器中找到所选方案的密钥，则将附加“未解析密钥”，因此该队列中的任何工作项操作都可能会失败。
与其他队列配置相同，只有在按下“应用”按钮后才会保存更改。
配置活动队列
队列可以配置为“活动”，这意味着它有一个分配给它的流程和一个资源组，为其提供处理其项所需的知识。
要使队列处于活动状态，请勾选“活动队列”复选框。将启用分配字段，且必须在保存更改之前选择流程和资源组。
请注意，在创建活动队列之前，必须存在资源组。
