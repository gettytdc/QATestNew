有关一般帮助，请参阅启动参数。

主题 1064—不要使用 DateDiff 函数的警告

应尽可能避免使用 datediff 函数，因为 Blue Prism 为实现类似目标提供了更好的方法。特别是，不应该用它来检测一天中的时间，因为给出的结果将视时间的夏令时变化而异。实现该目标的首选方法是将当前时间与已知时间的差异进行比较，以获得时间间隔；然后应将此时间间隔与固定值进行比较。

例如，表达式 

DateDiff(6, Today(), Now()) < (18*60)

旨在确定当前时间是否在下午 6:00 之前。首选方法如下 

LocalTime() - [MyReferenceTime] < [MyTimespan]

其中数据项“MyReferenceTime”表示今晚 6:00 的时刻，数据项“MyTimespan”存储值为零的时间间隔。

主题 8202—达到系统用户角色的最大数

主题 8212—输入的用户名无效

主题 32788—无法识别的应用程序类型

您的应用程序管理器版本不支持您的流程中指定的应用程序类型。您可以重新运行应用程序定义向导以选择新的应用程序类型，也可以从 Blue Prism 请求支持相关应用程序类型的已更新应用程序管理器版本。

要了解当前选择的应用程序类型，请将流程导出到本地 xml 文件，然后查找 xml 元素“ApplicationTypeInfo”。

主题 32798—应用程序未连接

您不能在目标应用程序启动之前与之交互。确保配置业务对象，以便在进行任何交互之前始终启动应用程序。

主题 32808—读取/写入/导航 Java Runtime Environment 版本 1.3.x 失败

Blue Prism 的 Java 自动化利用 Sun 的 Java Access Bridge 与目标应用程序进行交互。Sun 产品的一个已知限制是对 Java Runtime Environment (JRE) 版本 1.3 的支持不完整且容易出错。另请参阅 Java Access Bridge 故障排除页面。

主题 32818—未设置目标应用程序信息

在与目标应用程序交互之前，必须注册有关该应用程序的一些信息。 请访问应用程序建模器以填写此信息，然后保存您的业务对象。

主题 1048586—执行操作的权限不足

请参阅用户权限。

主题 524 298—操作由于许可限制而被拒绝

这意味着系统管理器中安装的当前许可证不允许执行所请求的操作。

请注意，如果您最近更改了许可证，则需要注销 Blue Prism 并重新登录（在每台计算机上）以使更改生效。

如果您认为有错误，请联系 Blue Prism 寻求技术支持。
