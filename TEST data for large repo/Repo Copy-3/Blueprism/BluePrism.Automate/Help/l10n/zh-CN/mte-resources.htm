多组环境

资源的访问权限

以下权限适用于资源。可以通过系统管理器，以及通过使用多组环境为用户角色配置这些权限。

权限

描述

作为资源通过身份验证

（仅适用于用户角色权限）

允许在启动资源时使用用户帐户进行运行时身份验证。不具备此权限的用户帐户将无法用于启动针对环境进行明确身份验证的资源。

从 6.3 之前的版本进行升级时，将向所有现有角色授予此权限以确保他们可继续启动现有资源。此权限不会自动授予到新安装中的新角色和所有角色，必须主动应用此权限。

当用户登录到 Blue Prism 客户端时，用户可以调试和启动可选择在本地自动启动的资源。
配置资源

用户可以在系统管理器中编辑资源，以重置资源的 FQDN，并更改资源的日志记录选项。如果用户也具有控制资源权限，则还可以撤回和取消撤回资源。查看资源和管理资源访问权限权限以隐式方式授予。对于相应配置，需具备此权限才能重置资源的 FQDN。

要将资源添加到活动工作队列，用户需至少具有控制资源权限所提供的针对相关资源组的访问权限。配置资源将不允许用户向活动队列添加资源
控制资源

用户可以在控制室中创建、启动、停止和删除会话。当正在运行会话时，具有此权限的用户可以在控制室中与该会话进行全面交互，以执行如启动流程或者发送终止或停止请求等任务。

具有此权限的用户将隐式地获得查看资源权限。
编辑资源组

用户可以在系统管理中创建、编辑、删除和重命名组。他们还可以配置资源层次结构，以及在组之间移动资源。

移动操作中涉及的任何受限制的组都需要“管理访问权限”和“编辑组”的权限。如需更多信息，请参阅移动组。
管理资源访问权限

如果在用户的角色中启用了管理访问权限，则用户还可以管理组的访问权限。具有此权限的用户无法为自己的用户角色更新该权限。这可以防止用户将自己的访问权限管理功能交给之前拒绝他们访问的组。查看资源权限是以隐式方式授予的。

查看资源

用户可以在控制室和系统管理中查看资源，并且可以查看与这些资源相关的会话管理数据。此权限还允许通过 telnet/HTTP 接口执行一些查询。

查看资源屏幕捕获

如果用户有权访问控制室，则还可以查看异常屏幕捕获。查看资源权限是以隐式方式授予的。

为了撤回资源，需拥有针对包含资源的组的“配置资源”、“编辑资源组”和“管理资源访问权限”权限。

资源角色

资源将继承控制用户的权限。在控制室中启动会话需要一组最小权限，这些权限在运行时资源角色中定义。如果您为具有较少权限的机器人用户使用角色，则可能会限制功能。

在未明确对环境进行身份验证的情况下启动的资源（不建议）会自动承担“运行时资源”角色。
