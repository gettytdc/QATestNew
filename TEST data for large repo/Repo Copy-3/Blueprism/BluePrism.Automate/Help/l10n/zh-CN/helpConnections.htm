“连接”对话框

“连接”对话框可用于存储不同的数据库连接配置。 您需要为每个连接分别指定一个名称；根据连接类型，您可能还需要提供其他信息。

连接类型

在下拉菜单中，共有五个连接类型可供选择：

  • Blue Prism 服务器
  • SQL Server（SQL 身份验证）
  • SQL Server（Windows 身份验证）
  • 可用性组（SQL 身份验证）
  • 可用性组（Windows 身份验证）

Blue Prism 服务器

连接到专用 Blue Prism 服务器是最为安全的一种连接配置。

请参阅 Blue Prism 服务器页面，以详细了解如何配置和运行服务器。

服务器名称

您必须提供正在运行 Blue Prism 服务器的计算机的主机名。本地计算机一定要能解析所提供的主机名。

连接模式

这可用于选择客户端与 Blue Prism 服务器之间的连接所要使用的连接模式。请注意，此设置必须与 Blue Prism 服务器配置中设置的相应值匹配。

端口

您必须提供 Blue Prism 服务器正在侦听的端口。

回调端口

指定客户端将要通过哪个端口侦听服务器回调操作。可以将此选项设置为 0，以便自动选择可用的端口。请注意，如果您通过此选项强制使用了某个特定端口，那么所有 Blue Prism 应用程序都只会有一个实例能从这台计算机连接到服务器。此设置仅适用于采用 .NET Remoting 连接模式的 Blue Prism 服务器连接。

SQL Server（SQL 身份验证）

“SQL 身份验证”会使用 SQL 用户的凭据来访问数据库。

所含的数据库在 Blue Prism 中受支持，但必须通过手动方式使用 AutomateC 中的 /getdbscript 选项所生成的脚本来创建。

服务器名称

您必须提供托管数据库服务器的计算机的主机名。本地计算机一定要能解析所提供的主机名。

数据库名称

您的 Blue Prism 数据库显示在数据库服务器中的名称。如果还不存在此类数据库，则可在此处选择一个数据库名称。然后，您可以按照创建数据库帮助页面所述，通过 Blue Prism 用户界面创建一个数据库。

数据库用户 ID

请在此处提供 Blue Prism 在访问数据库时应该使用的数据库用户的用户名。

数据库用户密码

请在此处提供上方字段中所指定的用户的密码。Blue Prism 绝不会泄露这个密码。

其他 SQL 连接参数（仅限非 BP 服务器）

应该附加到为访问数据库而创建的查询字符串中的任何其他参数（用分号分隔），例如 Encrypt=true;TrustServerCertificate=true.

SQL Server（Windows 身份验证）

“Windows 身份验证”会使用当前登录的 Windows 用户的凭据来访问数据库。 必须向该 Windows 用户授予数据库服务器的访问权限，以便其访问数据库。

服务器名称

您必须提供托管数据库服务器的计算机的主机名。本地计算机一定要能解析所提供的主机名。

数据库名称

您的 Blue Prism 数据库显示在数据库服务器中的名称。如果还不存在此类数据库，则可在此处选择一个数据库名称。然后，您可以按照创建数据库帮助页面所述，通过 Blue Prism 用户界面创建一个数据库。

可用性组

有两个可用性组选项与对应的 SQL Server 选项相同，但前者会连接到 SQL Server 可用性组。“服务器名称”可指定可用性组侦听器，且有两个额外设置可用：

端口

可用性组侦听器正在侦听的端口。

多子网故障转移

允许在连接上启用多子网故障转移。有关详细信息，请参阅 Microsoft 的文档。

使用多个连接

您可能会想要存储多个连接；例如，一个数据库用来进行测试，另一个数据库用来执行实时操作。这个对话框能让您从一个连接快速转移到另一个连接。新的连接可通过“新建连接”按钮来添加。然后，可在右侧填写新连接的相关详细信息。

要编辑现有连接，只需在左侧列表中将其选中，然后在右侧填写相关详细信息。无论选择的是哪个连接，只要按下“确定”按钮，所选连接就会成为 Blue Prism 使用的活动连接。

出于安全考虑，系统只会存储当前连接的密码，所以在从一个连接切换到另一个连接时，您必须重新输入相应的密码。
