Blue Prism 流程历史记录—概述

流程历史记录功能使 Blue Prism 管理员（或外部审计员）能够审核两次保存之间对流程所做的更改。每次保存后系统都会创建一个流程副本，并无限期地保留该流程版本。流程比较窗体允许通过并排放置两个流程并突出显示流程中已更改的环节来直接比较两个流程。

可以比较相同流程的两个版本（即编辑前和编辑后）；此功能如下所述。或者，也可以比较两个任意流程；具体描述请参阅流程比较帮助。

选择要审查的流程

管理员可以在“流程历史记录”选项卡下的“流程”部分中比较来自系统管理器的同一流程的两个版本。所有可用的流程都列在下拉菜单中，其中包括所有过去的流程，甚至已删除的流程。

选择一个流程后，下拉菜单下方的列表视图将填充该流程的所有版本以及该版本的创建时间和日期（即按下“保存”按钮的时间）、创建该版本的用户以及编辑摘要。

可以通过选择两个相关版本并点击“比较所选流程”来查看在两个版本之间所做的更改。

使用比较窗体

一旦完全加载，比较窗体将并排显示两个流程，并根据以下方案突出显示所有修改环节：新环节以绿色突出显示；已删除的环节以红色突出显示；已修改的环节以黄色突出显示。

通过将鼠标悬停在已修改的环节上，用户可以通过阅读黄色工具提示来发现（Blue Prism 认为的）该环节发生的最重大变化。通过右键点击相关环节并选择“显示完整更改列表”，可以看到该工具发现的详尽更改列表。此外，用户可以通过像在 Process Studio 中一样查看环节的属性来手动比较环节。

此窗体菜单中的选项（如打印、缩放等）适用于当前选定的流程版本。此选择由流程周围的蓝色带区指示。 如果要从流程比较工具打印流程版本，请点击要打印的流程版本的任何部分以选择该版本，确认该版本现在包围在蓝色带区中，然后从“文件”菜单中选择“打印”。
