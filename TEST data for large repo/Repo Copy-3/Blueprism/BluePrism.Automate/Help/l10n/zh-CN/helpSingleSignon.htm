帮助—单点登录

概述

Blue Prism 提供两种管理平台身份验证的方法：

  • 本机 Blue Prism 身份验证
  • Blue Prism 单点登录（推荐用于企业部署）
身份验证方案必须在创建数据库时选择—之后即无法更改。
可以配置针对不同数据库的连接，每个数据库可以实现不同的 Blue Prism 登录方法。

本机 Blue Prism 身份验证

默认情况下，Blue Prism 使用自己的身份验证机制。用户帐户在 Blue Prism 中单独创建和维护，处理用户登录尝试的方式是验证 Blue Prism 数据库中配置的用户名/密码组合。用户的各个权限和角色通过分配 Blue Prism 用户角色来进行维护。

Blue Prism 的单点登录

Blue Prism 支持使用 Microsoft Active Directory 域服务的单点登录，该服务允许经过操作系统身份验证且属于相应域安全组成员的用户登录 Blue Prism 而无需重新提供凭据。

单点登录为大量用户提供单点管理和访问控制，使系统管理员能够从中受益。

配置 Blue Prism 的单点登录

Blue Prism 的单点登录实施根据 Active Directory 安全组成员身份对用户帐户应用访问控制。

为 Blue Prism 配置单点登录身份验证时，必须指定将与 Blue Prism 安全角色关联的安全组所在的 Active Directory 域。此外，还必须选择其成员将被授予系统管理员访问权限的安全组。

一旦为系统管理员配置了访问权限，就可以在其他 Blue Prism 安全角色和 Active Directory 安全组之间进行映射。
