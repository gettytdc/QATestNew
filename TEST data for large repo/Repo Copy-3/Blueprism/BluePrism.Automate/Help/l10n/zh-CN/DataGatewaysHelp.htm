关于数据网关。

数据网关提供了一种易于使用的集中式方法，可将数据从 Blue Prism 中推出，供外部系统用于监控和报告、长期数据存储以及提供机器学习模型。利用高级配置方法，可以将数据定向到任何需要的目标。可以对数据进行可视化和分析，以提供有关 Blue Prism 环境的宝贵见解，而无需在每个相关流程自动化中手动构建类似功能。

通过提供在 Blue Prism 数据库之外存储数据的功能，组织可以使用数据网关支持灵活的数据存储要求。例如，组织可能希望将所有会话数据保存在 Blue Prism 数据库之外，或者他们可能选择将数据存储在数据库中较短一段时间，从而推出这些数据的副本以进行长期数据存储。

通过应用设置来确定 Logstash 提供支持的数据网关引擎将处理哪些数据，通过配置定义将数据推送到哪些目标输出。会话日志、已发布的仪表盘和流程环节中的数据可以发送到各种外部输出（HTTP 端点、外部数据库、第三方分析工具和平面文件），从而可灵活控制数据分析和存储。

先决条件

配置数据网关之前，必须满足以下条件：

·         Blue Prism 版本必须是 6.5 企业版或更高版本。

·         必须配置 Blue Prism 数据库所在的 SQL Server 实例，以允许数据网关引擎与之连接。这需要：

o   启用 TCP/IP 协议并将其设置为“侦听全部”

o   将 SQL Server 设置为侦听端口 1433。

o   将服务器身份验证配置为接受 SQL Server 和 Windows 身份验证两种模式。 必须先在 SQL Server 中创建登录凭据，数据网关才能访问 Blue Prism 数据库。

设置概述

配置数据网关需要以下步骤：

        1.       安装数据网关引擎组件

数据网关引擎要求在运行数据网关流程的所有应用程序服务器所在的环境中安装多个组件。

        2.       启用数据网关流程

必须在应用程序服务器上启用数据网关流程，并在每个应用程序服务器的“服务器配置详细信息”中选择用于与数据网关进行通信的端口。

        3.       创建数据网关 SQL 登录凭据和用户。

数据网关需要对 Blue Prism 数据库所在的服务器实例创建 SQL 登录凭据和用户。

        4.       添加保存了 SQL Server 用户凭据的数据网关凭据

必须使用 Blue Prism 凭据才能从数据网关引擎安全访问 Blue Prism 数据库。

        5.       配置 Blue Prism 数据网关设置

这些设置用于确定数据网关引擎将处理哪些数据。

        6.       创建数据网关配置

数据网关配置中的输出用于定义数据的目标发送位置。这可以是平面文件、HTTP 端点、Splunk 实例或数据库。

环境设置

安装数据网关引擎组件

数据网关要求在运行数据网关的所有应用程序服务器所用的环境中安装多个组件。如果应用程序服务器托管多个环境，则必须在将运行 Logstash 服务的每个环境中安装组件。

组件包含在另一个安装程序 Blue Prism 数据网关组件中，可从 Blue Prism 门户获得。

创建数据网关 SQL 登录凭据和用户

对 Blue Prism 数据库所在的服务器实例运行以下 SQL 代码。这将创建一个允许数据网关访问服务器的登录凭据和用户。

将 password 替换为一个复杂密码，以用于数据网关登录。

CREATE LOGIN Data Gateways BP Database SQL User WITH PASSWORD = 'password';

CREATE USER Data Gateways BP Database SQL User FOR LOGIN Data Gateways BP Database SQL User

GO

sp_addrolemember 'BPA_DataGatewaysEngine', 'Data Gateways BP Database SQL User'

GO

Blue Prism 设置。

数据网关凭据。

数据网关凭据可用于任何必须经过身份验证才能访问数据库或 HTTP 端点的数据网关配置。除此之外，还必须创建数据网关凭据来存储 SQL Server 用户账户，该帐户为数据网关引擎提供对数据库的访问权限。

“数据网关”凭据类型的凭据只能由应用程序服务器在生成数据网关配置时访问，它们不能用于标准 Blue Prism 流程或其他 Blue Prism 元素（如 Web API 定义）。

为 SQL Server 用户添加凭据

创建凭据来存储安全访问 Blue Prism 数据库所需的 SQL Server 用户帐户。

凭据名称、用户名和密码必须与下面指定的凭据名称、用户名和密码完全匹配，以便反映为 SQL Server 登录输入的详细信息。

        1.       选择“系统”选项卡。

        2.       选择“安全”>“凭据”。

        3.       点击“新建”。

        4.       配置以下凭据：

·         名称—数据网关 BP 数据库 SQL 用户

·         类型—数据网关凭据

·         用户名—数据网关 BP 数据库 SQL 用户

·         密码—为 SQL Server 用户指定的密码。

        5.       点击“确定”保存凭据。

配置数据网关设置

通过确定将处理哪些数据、在何处处理相关数据以及发送数据的频率来配置数据网关引擎。

        1.       选择“系统”选项卡。

        2.       选择“数据网关”>“设置”。

        3.       选择所需选项，确定会话日志数据存储的位置（至少需要选中一个选项）：

·         往数据库写入会话日志—会话日志将会发送至 Blue Prism 数据库中的会话日志表。如果未选择，则在 Blue Prism 客户端中查看会话日志的功能将不可用。

·         将会话日志发送至数据网关—会话日志将发送到 Blue Prism 数据库上的临时存储表，数据网关引擎将在其中访问它们，以便在配置的输出中使用。

        4.       选择“将已发布仪表盘发送至数据网关”，来将数据从已发布的仪表盘发送到数据网关系统中的数据库表，并设置为每个仪表盘发送数据的频率。

        5.       将“状态监控频率”设置成 5 到 3600 秒之间的值。这决定了“控制室”屏幕中“数据网关”屏幕的刷新频率。

有关 Blue Prism 仪表盘的更多信息，请参阅产品内帮助中的仪表盘概述主题。

        6.       点击“应用”以保存设置。

数据网关配置

数据网关配置是一个输出集合，用于定义会话日志数据、已发布的仪表盘和/或自定义对象发送到的位置。对于每个输出，可以将数据发送到文件、HTTP 端点、Splunk 实例或数据库。通过配置可以灵活支持各种需求：一种数据类型可以发送到多个目标，多种数据类型可以发送到单个目标。

将输出添加到数据网关配置

数据网关配置可以包含任意数量的单独输出。每个输出中的数据会添加到单个配置文件中。

        1.       点击“系统”选项卡并选择“数据网关”>“配置”。

        2.       点击“添加新的网关配置”。

        3.       系统会显示数据网关配置向导。

        4.       输入唯一的输出名称。

        5.       选择一种输出类型（文件、HTTP 端点、Splunk 或数据库），然后填写该输出类型的字段。

        6.       点击“下一步”。

        7.       选择将发送到输出的数据：

·         会话日志 – 数据网关将处理此输出类型的会话日志数据。

·         已发布的仪表盘 – 数据网关将处理此输出类型的已配置完毕的已发布仪表盘中的数据。

·         自定义对象数据 – 数据网关将处理来自配置为使用数据网关内部业务对象的任何 Blue Prism 操作的数据。

        8.       点击“下一步”。 此时将显示输出数据的预览。

通过点击“编辑高级输出”可以直接编辑生成的输出中的数据。有关高级配置的更多信息，请参阅高级输出。

        9.       点击“完成”以将输出保存至配置。

输出类型

文件

将所选数据导出到指定文件。 虽然数据网关配置向导中的创建输出仅支持将数据发送到 .txt 文件，但可以编辑高级输出和自定义配置，以支持其他文件格式（例如 .csv）。

有关如何实现此目的的详细信息，请参阅配置文件结构。

动态文件路径可用于在使用年、月、日和时间变量的文件名中包含创建日期。 例如：C:\Session Logs\session_logs_%{+YYYY-MM-dd}.txt

这将每天在指定位置创建一个新的文本文件，其文件名包含当前日期：session_logs_2019-04-30.txt

也可以在文件路径中使用变量来创建基于日期的文件夹名称。例如：

C:\Session Logs\%{+YYYY}\%{+MM}\%{+dd}\session_logs_%{+YYYY-MM-dd-hh}.txt

这将根据当前日期创建文件夹并附加文件名。例如：

C:\Session Logs\2019\04\30\session_logs_2019-04-30-11.txt

使用动态文件路径时生成的文件/文件夹时间戳使用 UTC 格式。

动态文件路径可用在使用数据网关向导创建的配置和高级配置中。

HTTP 端点

使用所选方法通过指定的 HTTP 端点路由数据。如果需要身份验证才能访问端点，请选择数据网关凭据。

Splunk

使用关联的 API 令牌将所选数据发送到指定的 Splunk URL。

数据库

将所选数据发送到指定的数据库表。如果需要身份验证才能访问数据库，请选择数据网关凭据。

高级输出

创建或编辑输出时，具有数据网关 – 高级配置权限的用户可以直接编辑输出中的代码以创建高级配置。

在输出向导的预览屏幕中 – 单击编辑高级配置以打开高级编辑器。

可以通过输入数据或从剪贴板粘贴数据来编辑输出。还可将文件（最大 10k）导入到输出，替换所有当前数据。完成所需的更改后，点击“保存高级输出”。

将输出保存为高级输出后，即使未进行任何更改，也无法再在“数据网关配置”向导中进行配置—只能在高级编辑器中进行配置。

 

管理数据网关配置

“数据网关”>“配置”屏幕中列出了数据网关配置的输出。点击“管理”可复制、编辑或删除配置。

每当更改数据网关配置时，无论是新的、已更新的还是已删除的输出，都必须重新启动数据网关引擎。有关更多信息，请参阅启动和停止数据网关引擎。

复制输出

点击与要复制的输出对应的“复制”。系统将根据输出类型，打开数据网关向导或高级编辑器，以便根据需要更新输出。保存输出后，重新启动数据网关流程。

删除输出

点击与要删除的输出对应的“删除”，以将其从配置中删除，然后重新启动数据网关流程。

编辑输出

点击与要编辑的输出对应的“编辑”。 系统将根据输出，显示输出向导或高级编辑器，以便更新输出。保存输出后，重新启动数据网关流程。

自定义配置

具有数据网关 – 高级配置权限的用户可以通过直接编辑基础代码来创建自定义配置。这允许用户创建具有超出向导所提供功能的配置，例如仅将特定事件字段发送到输出，或自定义发送到输出的数据的格式。

有关配置文件组成的更多信息，请参阅配置文件结构。

创建自定义配置

        1.       从数据网关 – 高级配置中单击查看配置

此时会显示配置代码。

        2.       点击“编辑配置”，使配置代码处于可编辑状态。

        3.       进行必要的更新后，点击“保存”并确认。

必须重新启动数据网关引擎才能使更改生效。有关更多信息，请参阅启动和停止数据网关引擎。

自定义配置将覆盖现有配置中的输出。这可在数据网关屏幕中看出，其中高级配置代码将替换输出列表。

用户可随时点击自定义配置中的“删除”，将配置还原为其原始组成并显示原始输出。

 

在控制室中管理数据网关流程。

控制室中列出了数据网关流程，您可以从中查看状态并重新启动流程。

选择“控制”选项卡并突出显示“数据网关”节点，以显示环境中的所有数据网关流程。

流程可以处于以下状态：

·         联机—托管数据网关流程的应用程序服务器处于联机状态。

·         脱机—托管数据网关流程的应用程序服务器处于脱机状态。

·         正在启动—数据网关流程正在启动。

·         正在运行—数据网关流程正在处理事件。

·         错误—数据网关流程遇到错误。它将尝试定期重启以纠正错误。

·         无法恢复的错误—数据网关流程遇到无法通过重启予以修复的错误。它不会尝试重启。

启动和停止数据网关引擎

右键点击流程以打开上下文菜单。 根据流程的当前状态，视需要选择“启动”或“停止”。

数据网关 VBO

数据网关内部业务对象可以在任何流程的操作环节中使用，并可配置为在流程运行时将数据从集合发送到数据网关引擎。可以使用 VBO 将任何可以放入集合的数据发送到数据网关引擎。

VBO 具有一个操作，即发送自定义数据，该操作可接受集合作为输入，并将这些集合中的数据发送到数据网关引擎。使用 VBO 的所有操作中的数据将供配置为发送自定义对象数据的任何输出使用。

配置用户角色权限。

可以将三种数据网关用户权限授予相应的管理员用户角色：

·         数据网关 – 配置 – 用户可以配置数据网关设置，并添加和管理网关输出。他们无法创建或编辑高级输出或自定义配置。

·         数据网关 – 高级配置 - 用户可以配置数据网关设置，并添加和管理网关输出。他们可以创建和编辑高级输出和自定义配置。此权限只应授予具有编辑输出和配置代码知识的专家用户。

·         数据网关—控制室—用户可以访问控制室中的数据网关节点，以查看数据网关引擎的状态，并根据需要启动和停止流程。

这些权限将授予系统管理员，以方便其执行升级和新的安装—它们不会针对任何其他内置用户角色启用。

高级设置

此部分提供了只应由有经验的用户执行的配置和编辑方法的详细信息。

配置文件结构

配置文件是根据已配置的数据网关设置和输出创建的。如下所述，它们由三个主要部分组成：输入、输出和筛选器。

输入

此部分由 Blue Prism 根据 Blue Prism 数据库设置自动生成。它决定如何将事件置入数据网关引擎以进行处理。在下面的示例中，它们是从 Blue Prism 数据库中的 BPADataPipelineInput 表中检索的。

input {

jdbc {

jdbc_driver_library => "..\sqljdbc_4.2\enu\jre8\sqljdbc42.jar"

jdbc_connection_string => "jdbc:sqlserver://SQL_SERVER_INSTANCE:1433;databaseName=BP_DATABASE;"

jdbc_driver_class => "com.microsoft.sqlserver.jdbc.SQLServerDriver"

jdbc_user => "<%Data Gateways BP Database SQL User.username%>"

jdbc_password => "<%Data Gateways BP Database SQL User.password%>"

statement => "delete top(3000)from BPADataPipelineInput with (rowlock, readpast) output deleted.eventdata"

schedule => "*/3* * * * *"

}

}

如果需要，可以对以下输入区域进行编辑，以满足所需首选项的要求。

 

`schedule => "*/3* * * * *"`

这可决定 SQL 查询从 BPADataPipelineInput 表请求数据的频率。 可将默认值（每三秒一次）更新为所需的值。

 

`statement => "delete top(3000)from BPADataPipelineInput with (rowlock, readpast) output deleted.eventdata"`

这是针对 Blue Prism 数据库执行的 SQL 语句，用于从 BPADataPipelineInput 表中提取事件。该值用于控制每个时间间隔从 BPADatapipelineInput 表中提取的最大行数。如果需要，可以更改默认值 (3000)。

筛选器

筛选器可用于对事件执行中间处理。这些可以是在将事件的某些字段发送到输出之前对其执行的添加、删除或修改等操作（例如删除会话日志的 AttributeXML 字段）。

以下网址中列出了所有可用的筛选器插件列表：https://www.elastic.co/guide/en/logstash/current/filter-plugins.html

默认情况下，Blue Prism 生成的配置将包含一个筛选器：

filter{

json {

source => "eventdata"

target => "event"

}

}

默认情况下，配置包含单个 JSON 过滤器，用于解析和扩展包含已配置数据类型（会话日志、仪表盘等）的 JSON 字符串，以便可以在配置文件中访问内容。

可以编辑此部分以添加和删除筛选器，但不应删除或修改默认的 JSON 筛选器。

输出：

输出决定事件发送到的位置。如果已使用数据网关向导配置了输出，则它们会在此处列出。处理的每个事件都将发送到配置中列出的每个输出。

output {

file {

path => "C:\data.txt"

}

 

csv {

path => "C:\data.csv"

}

}

在上面的示例中，指定了 .txt 文件和 .csv 输出。发送到数据网关系统的每个事件都将写入文本文件 C:\data.txt 和 csv 文件 C:\data.csv

有关可用输出的列表，请参见此处：https://www.elastic.co/guide/en/logstash/current/output-plugins.html

 

事件结构

此部分详细介绍了在从 Blue Prism 收到事件后，这些事件在 Logstash 中的结构。此信息可用于在 Logstash 配置中构造条件语句，以根据内容将事件转移到输出，或者为输出创建自定义消息格式。

事件（会话日志或已发布的仪表盘）会作为 JSON 字符串存储在 Blue Prism 数据库中。要将此 JSON 字符串转换为可以在 Logstash 中使用的一组字段，可将 JSON 筛选器添加到配置中：

filter{

json{source => "eventdata"

target => "event"}

}

这会将会话日志/已发布的仪表盘添加为嵌套在“事件”字段下的字段。

例如——

[event][eventType] 包含事件类型（会话日志、已发布的仪表盘或自定义对象数据）。

[event][EventData] 包含作为嵌套字段的事件数据。

[event][EventData][SessionNumber] 包含会话编号（如果这是会话日志事件）。

要仅将名为“ProcessA”的流程中的会话日志发送到文本文件，可以为输出使用条件语句：

output {

If [event][eventType] == 1 and [event][EventData][ProcessName] == “ProcessA” {

file {

path => “C:\log.txt”

}

}

}

有关可用字段的完整列表，请参阅下表。

常规

事件

描述

[event][eventType]

表示事件类型的编号：

1 = 会话日志

2 = 已发布的仪表盘

3 = 自定义

[event][EventData]

事件数据。此类数据的结构将因事件类型而异。

事件类型—会话日志

事件

描述

[event][EventData][StartDate]

使用 ISO 8601 表示法格式化的流程环节的开始日期。例如："2019-02-11T07:59:54.829674+00:00"

[event][EventData][SessionNumber]

此会话日志所属会话的会话编号。

[event][EventData][ResultType]

流程环节的结果类型。

[event][EventData][Result]

流程环节的结果。

[event][EventData][AttributeXML]

序列化为 XML 的环节的输入和输出参数。

[event][EventData][ProcessName]

此环节所属的业务对象的名称。如果通过流程记录会话日志，则此值为空。

根据内容将数据定向到输出。

在将会话日志和仪表盘数据发送到单独的文本文件时，可将条件语句应用于输出，只有当输出满足其中的一个或多个条件时，系统才会将事件传递给输出。这可让在高级编辑器中自定义或在外部文本编辑器中创建的输出支持 Logstash 功能（此功能在数据网关配置向导中并未提供）。例如，可以编辑输出以仅发送指定流程或仪表盘磁贴的数据。

在此示例中，文件输出的条件语句检查某个 EventType 值。 会话日志的事件类型为 1，仪表盘的事件类型为 2。

input {

jdbc_driver_class => "com.microsoft.sqlserver.jdbc.SQLServerDriver"

jdbc_user => "<%Data Gateways BP Database SQL User.username%>"

jdbc_password => "<%Data Gateways BP Database SQL User.password%>"

statement => "delete top(3000)from BPADataPipelineInput with (rowlock, readpast) output deleted.eventdata"

schedule => "*/3* * * * *"

}

}

filter {

json{source => "eventdata"

target => "event"}

}

output {

if [event][EventType] == 1 {

file { path => "C:\sessionlogs.txt" }

}

if [event][EventType] == 2 {

file { path => "C:\dashboardlogs.txt"}

}

}

自定义配置中的凭据。

如果配置中需要凭据或其他敏感数据，则应将其添加到 Blue Prism 凭据，然后按凭据名称在配置中进行引用。

创建用于数据网关配置的 Blue Prism 凭据时，凭据类型必须是数据网关凭据。这些凭据只能由数据网关系统访问，Blue Prism 流程无法访问。

可以使用语法 <<%{credentialname}.{property}%>> 在配置中引用凭据，其中 {credentialname} 是凭据的名称，{property} 是凭据中属性的名称。

例如，要使用名为 cred1 的凭据的用户名，配置代码应分别为 <<%cred1.username%>> 和 <<%cred1.password%>>。

可以使用属性名称访问凭据自定义属性。

故障排除。

如果数据网关系统未按预期工作，以下信息可帮助您查明问题。

在控制室中查看数据网关流程的状态。

如果它处于错误状态，状态消息将提供与问题相关的信息。

在应用程序服务器上查看事件日志。

任何数据网关流程错误都将从 Logstash 流程管理器源输出到事件日志中。Logstash 流程的标准输出流也会在发生错误时写入事件日志，这有助于识别 Logstash 相关错误。

在应用程序服务器上查看 Logstash 日志。

这些日志会写入 C:\Logstash\logstash\logs\。
