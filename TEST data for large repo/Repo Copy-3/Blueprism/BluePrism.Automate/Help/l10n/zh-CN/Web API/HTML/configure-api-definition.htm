创建 Web API 定义

Web API 定义包含指示 Blue Prism 如何向 Web API 发出 HTTP 请求所需的所有数据。API 定义中的数据确定每个请求的构造方式，并指定发出请求所需的 URL、HTTP 方法、标头和身份验证。API 定义还允许定义可用于将数据传递到请求的参数。

每个 API 定义由以下元素组成：

  • 基本设置—用于在 API 定义中配置的所有 API 请求的名称、基本 URL 和超时设置。
  • 操作—Web API 提供的将供 Blue Prism 使用的功能。这些操作详细说明了定义中每个 API 请求的请求正文和响应处理。
  • HTTP 标头—将在此 API 定义的请求中使用的标头。所有请求中都使用通用标头，而且，在定义了特定于操作的标头的操作中，除了使用通用标头外，还会使用特定于操作的标头。
  • 参数—以动态方式扩展引用，这些引用用于将数据传递到基本 URL、URL 路径、标头和请求正文。参数也可以在“主题”字段中用于 OAuth 2.0（JWT 持有者令牌）配置。通用参数可用于任何操作，而特定于操作的参数只能用于定义该参数的操作。
  • 通用身份验证—对 API 的每个请求进行身份验证的方法。
  • 通用代码—用于创建复杂 API 请求和响应的代码。

Web API 定义是在 Blue Prism 交互式客户端的“系统”设置中配置的。

  1. 选择系统选项卡。
  2. 从导航树中选择“对象”>“Web API 服务”。

  3. 从选项菜单中点击“添加服务”。

    系统将显示“新建 Web API 服务”对话框。
