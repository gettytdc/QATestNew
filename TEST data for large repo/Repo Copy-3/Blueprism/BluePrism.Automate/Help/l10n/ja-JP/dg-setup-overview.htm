設定の概要

データゲートウェイの構成には次のステップが必要です。

  1. データゲートウェイエンジンコンポーネントをインストールする

    データゲートウェイエンジンでは、データゲートウェイプロセスを実行しているすべてのアプリケーションサーバーがある環境に、多数のコンポーネントをインストールする必要があります。

  2. データゲートウェイプロセスを有効にする

    データゲートウェイを実行するために必要な各アプリケーションサーバーの［サーバー構成の詳細］で、データゲートウェイプロセスを有効にし、データゲートウェイプロセスとの通信に使われるポートを選択する必要があります。

  3. データゲートウェイSQLログインとユーザーを作成する

    データゲートウェイでは、Blue Prismデータベースが配置されるサーバーインスタンスに対し、SQログインおよびユーザーを作成する必要があります。

  4. SQL Serverのユーザー認証情報を保持するデータゲートウェイ認証情報を追加する

    Blue Prismデータベースにデータゲートウェイエンジンからセキュアにアクセスするには、Blue Prismの認証情報が必要です。

  5. Blue Prismデータゲートウェイ設定を構成する

    これらの設定によりデータゲートウェイエンジンによって処理されるデータが決まります。

  6. データゲートウェイ構成を作成する

    データゲートウェイ構成の出力は、データの送信先を定義します。これは、フラットファイル、HTTPエンドポイント、Splunkインスタンス、データベースのいずれかになります。
