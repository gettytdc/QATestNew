リソースのアクセス権

次の許可はリソースに対して使用できます。また、システムマネージャーのユーザーの役割に対して、Multi-Team Environmentsを使用して構成できます。

許可

説明

リソースとして認証

（ユーザーの役割の許可としてのみ利用可能）

リソースを開始するとき、ランタイム認証に対してユーザーアカウントを使用できます。この許可を持たないユーザーアカウントでは、環境に対して明示的に認証するリソースを開始することはできません。

6.3より前のバージョンからアップグレードするとき、既存のリソースを引き続き開始できるよう、既存のすべての役割にこの許可が与えられます。新しい役割と新しいインストールのすべての役割にこの許可が自動的に付与されることはありません。意図的に適用する必要があります。

Blue Prismクライアントにログインしたとき、オプションとして自動的にローカルで実行されるリソースをユーザーはデバッグしたり、開始したりできます。
リソースを構成

ユーザーはシステムマネージャーのリソースを編集し、リソースのFQDNをリセットしたり、リソースのログオプションを変更したりできます。ユーザーは、コントロールリソースという許可も与えられていれば、リソースを廃止したり、回収したりすることもできます。リソースを表示許可とリソースアクセス権を管理許可は暗黙的に与えられます。適切な構成では、リソースのFQDNをリセットするにはこの許可が必要になります。

Active Work Queueにリソースを追加するには、少なくともコントロールリソース許可によって与えられる関連リソースグループに対するアクセス権がユーザーに必要になります。リソースを構成では、ユーザーはアクティブキューにリソースを追加できません。
リソースをコントロール

ユーザーはコントロールルームでセッションを作成、開始、停止、削除できます。セッションが実行されると、この許可が与えられたユーザーは、コントロールルームでセッションを完全に操作でき、プロセスの起動や強制終了要求または停止要求の送信などのタスクを実行できます。

この許可が与えられたユーザーにはリソースを表示許可が暗黙的に与えられます。
リソースグループを編集

ユーザーはシステム管理でグループを作成、編集、削除、名前変更できます。また、リソース階層を構成したり、グループ間でリソースを移動したりできます。

移動に関与するあらゆる制限付きグループには、アクセス権の管理許可とグループの編集許可の両方が必要になります。詳細については、「グループを移動する」を参照してください。
リソースアクセス権を管理

自分の役割で［アクセス権を管理］が有効になっている場合、ユーザーはグループのアクセス権を管理できます。この許可が与えられたユーザーは自分自身のユーザーの役割のアクセス権を変更することはできません。そのため、ユーザーは以前にアクセスできなかったグループへのアクセス権を自分に付与することはできません。リソースを表示許可は暗黙的に与えられます。

リソースを表示

ユーザーはコントロールルームとシステム管理でリソースを表示できます。また、そのリソースに関連するセッション管理データを表示できます。この許可があれば、telnet/HTTPインターフェイスから照会することもできます。

リソーススクリーンキャプチャを表示

ユーザーはコントロールルームにアクセスできる場合、例外の画面キャプチャを表示できます。リソースを表示許可は暗黙的に与えられます。

リソースを廃止するには、そのリソースが含まれるグループで［リソースを構成］許可、［リソースグループを編集］許可、［リソースアクセス権を管理］許可が必要になります。

リソースの役割

リソースはコントロールユーザーの許可を継承します。コントロールルームでセッションを開始するには最小限の許可が必要になります。これはランタイムリソースの役割に定義されています。許可の少ないロボットユーザーの役割を使用している場合、機能が制限されることがあります。

環境に対して明示的に認証せずに開始（推奨されません）したリソースにはランタイムリソースの役割が自動的に与えられます。
