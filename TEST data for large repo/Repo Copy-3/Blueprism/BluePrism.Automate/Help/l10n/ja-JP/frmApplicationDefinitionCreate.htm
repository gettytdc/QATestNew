アプリケーションモデラー

アプリケーションパラメーターウィザード

はじめに

オブジェクトスタジオでビジネスオブジェクトを作成するときの最初のタスクは、通常、対象アプリケーションを定義することです。これには、対象アプリケーションに関する情報をBlue Prismに提供し、アプリケーションモデラーでアプリケーション定義を構築することが含まれています。

あるいは、対象アプリケーションが別の共有可能なビジネスオブジェクト内ですでに定義されている場合は、そのオブジェクトを親として選択することでそのモデルを使用できます。

ウィザードにアクセスする

以前にウィザードを実行したことがない場合は、アプリケーションモデラーを初めて開くときに表示されます。これを行うには、オブジェクトスタジオツールバーの適切なボタンを押します。あるいは、アプリケーションモデラーを開き、アプリケーション定義ツリー構造のルートノードを選択し、ツリーの反対側のペインから適切なボタンをクリックすることで、もう一度ウィザードを起動することもできます。

ウィザードを完了する

アプリケーションタイプを選択する

最初のタスクはアプリケーションタイプを選択することです。アプリケーションタイプの名前は、確実に選択できるように分かりやすい名前にしてください。迷った場合は、［Windowsアプリケーション］オプションを選択してください。

アプリケーションタイプによっては、ライセンスの制限を受けることがあります。これにより選択が制限され、最適なアプリケーションタイプを選択できなくなる可能性があります。このような状態になった場合は、Blue PrismPrismまでお問い合わせください。

詳細情報を提供する

アプリケーションタイプの選択の後には、通常、一連の質問が続きます。後に続く質問は、通常、前の質問に対する回答、特にアプリケーションタイプによって異なります。総じて、この情報は、対象アプリケーションの性質とそのランタイム環境を詳述する一連のパラメーターとして知られています。個々のパラメーターに関する情報や、有効な値の範囲の説明については、パラメーターのヘルプを参照してください。
