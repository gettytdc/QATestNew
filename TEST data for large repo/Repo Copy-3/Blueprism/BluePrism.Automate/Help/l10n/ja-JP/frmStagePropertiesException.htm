ステージタイプ

この機能はバージョンの制限を受けます。詳細については、バージョンのページを参照してください。

例外ステージ

例外ステージでは、プロセスは例外をレポートして実行を停止できます。 例外の概要については、以下を参照してください。例外のヘルプ

1.名前を入力

入力した名前によって例外が識別され、プロセス図に表示されます。

2.説明を入力

追加の説明フィールドには、さらに長い説明を入力できます。

3.例外タイプを入力

例外タイプは、例外の原因を一般化するために使用します。

  • ドロップダウンから既存の例外タイプを選択します。
  • テキストを入力して新しい例外タイプを作成します。

4.例外の詳細を入力

プロセスから追加の値を取得するために例外の詳細を使用できます。

  • 例外の詳細を任意の有効なBlue Prism式にできます。詳細については、「計算と決定」を参照してください。
  • なんらかのテキストが必要な場合は、そのテキストを引用符で囲みます。

5.現在の例外タイプと詳細を保持

これにより、例外タイプと詳細を現在の例外で再利用できます。再利用しない場合は、ExceptionType()式とExceptionDetail()式を使用してキャプチャする必要があります。たとえば、クリーンアップを実行しながらも例外を追加の回復ステージにプロパゲートできます。

6.画面キャプチャを保存

例外が発生したときに画面の画像が取得され、Blue Prismデータベースに保存されます。チェックボックスを有効にするには、システム設定で画面キャプチャ機能を有効にする必要があります。ランタイムリソースに保存されるのは最新の画面キャプチャのみです。 画面キャプチャが作成されるのはコントロールルームで実行されているプロセスのみです。プロセスのデバッグ中は画面キャプチャを行いませんが、画面キャプチャをいつ行ったかを示すメッセージがログに記録されます。
