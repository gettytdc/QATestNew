System Manager

Font Repository

Overview

Fonts available for use by the Font Recognition functionality can be stored and managed in the database using this System Manager component.

Importing Fonts

Fonts can be imported from XML files. Multiple fonts can be imported at one time by simply selecting all the files you want to import.

When a font is imported, its name is taken from the name of the file. If the name of the file begins with 'font_' then that part is ignored. Thus, the files "font_System.xml" and "System.xml" would both result in the import of a font named 'System'.

Exporting Fonts

To export a single font, select it and click 'Export'. Then choose/select an appropriate location and filename. The font is then exported in a format that can be imported again later.

To export multiple fonts, select them and click 'Export'. Then choose an appropriate directory, and all the selected fonts will be exported there as individual files. The files are named "font_Name.xml". This allows them to be easily recognised, while ensuring that they retain their original names when imported again later.

Deleting Fonts

For fonts that are no longer required in the database, simply select them and click 'Delete'.
