Blue Prism

Process Utilisation

BPDS_ProcessUtilisationByHour

Returns utilisation throughout yesterday broken down by process, in 2 hour intervals, i.e. the amount of time that the resources have spent actively running sessions.

Source: Blue Prism Database – Live tables and MI Repository (requires compiling of MI statistics to be enabled)

Sample Chart

Parameters

Parameter Name Data Type Description Validation
@BPProcessName Text The name of an individual process to be queried (if not specified all processes will be queried) None.
@DisplayUnits Text

Possible values are:

  • Second – values are returned in units of seconds
  • Minute – values are returned in units of minutes
  • Hour – values are returned in units of hours

  • Percentage – values are returned as a percentage of the total processing time available (e.g. if processing has been carried out by 5 resources on a given day then total processing time per interval would be 5 x 2 (resources x hours) = 10 processing hours. Therefore a result of 50% would suggest that 5 hours of processing time were achieved in the given time frame)

If not specified the utilisation values are returned in minutes
