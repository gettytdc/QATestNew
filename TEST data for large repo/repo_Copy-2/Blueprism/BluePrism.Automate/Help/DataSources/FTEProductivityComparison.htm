Blue Prism

FTE Productivity Comparison

BPDS_FTEProductivityComparison

Returns the number of cases completed over the specified period, optionally compared against metrics representing a human workforce. Note that parameter values are required to enable comparisons to be made. Data is only returned for completed days (i.e. cases completed after the the MI statistics were last compiled are not included).

Source: Blue Prism Database – Live tables and MI Repository (requires compiling of MI statistics to be enabled)

Parameters

Parameter Name Data Type Description Validation
@BPQueueName Text The name of an individual queue to be queried (if not specified all queues will be queried) None.
@NumberOfMonths Integer Reporting period in months (if not specified the number of months is 6) Max. 12 months
@FTEProductivity Decimal

Represents the daily productivity of one FTE for comparison (i.e. the average number of cases one FTE would work in a day). If not specified then the volume of completed cases is returned

 None.
@FTECost Decimal

Only applicable if @FTEProductivity has been specified, represents the fully loaded daily cost of one FTE for comparison

 None.
@DisplayAs Text

Only applicable if @FTEProductivity has been specified, possible values are:

  • Percentage – values are returned as a percentage of FTE productivity
  • Number – values are returned as the number of equivalent FTEs
  • Cost – values are returned as cost savings based on the equivalent number of FTEs (requires a values for @FTECost)

If not specified values are returned as a percentage of FTE productivity

 Percentage, Number or Cost
