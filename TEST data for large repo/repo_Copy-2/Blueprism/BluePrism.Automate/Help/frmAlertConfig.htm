Process Alerts Configuration

This page describes how users can amend their preferences for process alerts. You may wish to read the Process Alerts overview first.

Opening the Configuration Window

The configuration window can be opened either from the context menu of the task tray icon (see Process Alerts overview for details), or from System Manager. The former method allows the current user's settings to be changed only; the latter allows a system administrator to change the settings of any user.

Choosing Processes

At the top of the configuration form is a list of Blue Prism Processes. Note that only published processes will appear in this list. You should place a tick beside each process for which you are interested in receiving alerts. You may choose as few or as many processes as you please.

Choosing Alert Types

You may choose to be alerted when a variety of process-related events occur. The available events of interest are as follows.

Pending
When a pending session is created
Running
When a process starts
Complete
When a process completes
Terminated
When a session is terminated, due to an unhandled exception in the process
Stopped
When a process is stopped
Stage
When a process reaches an alert stage

You may choose to receive a particular alert type by placing a tick in the appropriate box.

Choosing Alert Methods

You may choose to be alerted via one or more methods. They are as follows:

Popup
A transient notification window which briefly appears at the corner of the screen.
Show History
The config window appears with the history tab focused, and remains on the screen until the user closes it.
Notification Icon
The icon in the windows task tray changes appearance, from a red bell (normal) to an orange bell (indicating that an alert has arrived). The details of the alert become visible when the user double clicks the icon.
Sound
Blue Prism will play a sound each time an alert is received. This is subject to the correct hardware being installed on your computer (including sound card and speakers) and the volume being audible.

You may choose to be alerted by one or more methods by placing a tick in the appropriate box(es).

Alert History

A history of process alerts can be viewed on the process alert configuration form. The current day's alerts are displayed by default and the user can navigate to previous days as required. An export facility also allows a day of alerts to be saved to file. Alert history will be maintained until the user archives the related session logs. Note however that unlike archived log data, alert history can not be restored once the session files have been archived.
