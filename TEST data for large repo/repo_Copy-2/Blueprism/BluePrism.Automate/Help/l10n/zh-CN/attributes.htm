应用程序属性—Win32 模式

简介

以下是 Windows 探测工具在 Win32 模式中返回的属性列表。有关由其他模式返回的属性列表，请参阅：—

  • Active Accessibility 模式
  • UI Automation 模式
  • Java 模式
  • HTML 模式

应用程序属性

WindowText
窗口中的文本。请注意，窗口可以是从应用程序窗口到按钮再到组合框项的任何内容。 因此，在考虑该属性时，必须考虑上下文。例如，按钮的 WindowText 是按钮上显示的文本；编辑字段的 WindowText 是已输入到该字段中的文本。
ClassName
为控件类型指定的名称。
CtrlID
拥有元素的应用程序提供给该元素的 ID。 并非每个应用程序都使用此功能。
序数
目标应用程序创建元素的顺序。例如，序数为 1 表示在目标应用程序中检测到的所有元素中首先创建了该元素。
X
元素相对于元素父窗口的 X 位置（即从左侧测量的水平位置），以像素坐标表示。请注意，此父窗口不一定是顶级窗口，但也可能是其他类型的容器。
Y
元素相对于元素父窗口的 Y 位置（即从顶部测量的垂直位置），以像素坐标表示。请注意，此父窗口不一定是顶级窗口，但也可能是其他类型的容器。
宽度
以像素为单位的元素宽度。
高度
以像素为单位的元素高度。
可见
指示元素是否可见。元素可能存在，但不可见，因此在等待环节执行 CheckExists 查询时要小心。
已启用
指示元素是否可用于目标应用程序中的交互。通俗来讲，这个概念也可以描述为元素是否“灰显”或“可点击”。按钮是根据情况发现已启用或禁用的元素的常见示例。
活动
指示窗口是否“处于焦点”。活动窗口是处理任何键盘输入的窗口。 通常一次只有一个活动窗口。另请参阅导航环节上的 ActivateApplication 命令。
ChildCount
指示元素包含多少个子窗口。请注意，这包括不可见的窗口、小尺寸的窗口和其他窗口遮挡的窗口。 因此，指示的数量可能与屏幕上可见的数量不符。

父属性

pOrdinal、pX、pWidth、pClassName 等
按照惯例，所有属性名称都以大写字母开头。此规则的一个例外与父属性有关。元素父窗口的属性是通过在属性名称前面放置一个小写的 p 来命名的。例如，pOrdinal 是指元素父窗口的序数。
