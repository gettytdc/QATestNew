管理者権限

特定の役割で保護されている管理アクションがいくつかあります。これらの役割を持つユーザーは、グループメンバーシップに関係なく、システム全体で特定のレコードタイプにアクセスする必要があります。したがって、これらの管理者権限を持つユーザーは、グループレベルのアクセス権内で構成されている範囲を超えたアクセス権を持つ可能性があります。

たとえばユーザーがシステム - アーカイブ許可を持つ場合は、セッションデータが関係しているプロセスまたはリソースに対する許可をユーザーが持っているかどうかに関係なく、すべてのセッションに対してアーカイブアクティビティを実行できます。

次の管理者権限はグループレベルで設定された許可の影響を受けません。

  • リリース - パッケージを閲覧するとき、ユーザーはグループに適用されている許可に関係なく、すべてのオブジェクトとプロセスを見ることができます。ただし、パッケージを構成してリリースパッケージを生成するときには制限が適用されます。
    • パッケージの作成 - ユーザーはアクセス権のないアイテムを含めることはできません。
    • パッケージの変更 - ユーザーはアクセス権のないアイテムを含むパッケージを変更できます。ただし、それらのアイテムは許可がないためアクセスできない状態であると同時に、削除されます。ユーザーがそれらのアイテムを再度追加することはできません。
    • リリースの作成 - ユーザーはアクセス権のないアイテムが入っているパッケージが含まれているリリースをエクスポートすることはできません。
  • ［システム］>［アーカイブ］ - アーカイブはグループに適用されている許可の影響を受けません。
  • ［システム］>［レポーティング］ - レポーティングはグループに適用されている許可の影響を受けません。
  • アナリティクス - ダッシュボードタイルに表示されるデータは、グループに適用されているアクセス権によってフィルタリングされません。
