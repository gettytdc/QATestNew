ステージタイプ

遷移ステージ

遷移ステージでは、対象アプリケーション内でナビゲーション手順（またはアクション）を実行します。例としては、ボタンをクリックする、メニューを開く、ダイアログを開く、ウィンドウを閉じる、キーストロークを送信するなどがあります。

プロパティのフォーム

［遷移プロパティ］フォームには、上下に2つのリストがあります。最初のリストは、実行されるナビゲーションアクションのリストです。これらのアクションは上から下に順番に実行されます。たとえば、1つ目のアクションは［ファイル］メニューを開くこと、2番目のアクションは［保存］をクリックすることです。ここでの順番は重要であるため、ナビゲーションアクションを正しい順番で追加するように注意する必要があります。

各ナビゲーションアクションは、アプリケーション要素とアクションから構成されています。アプリケーション要素はアクションの対象（クリックされるメニュー項目など）であり、アクションはその要素に対して行われる内容（例：クリック、開く、最小化など）です。

下のリストは、上のリストで現在選択されているナビゲーションアクションに対する一連の入力を表します。たとえば、アクションがクリックの場合は、そのクリックアクションの座標を指定する必要があります。このリストは動的であり、現在の行で選択されているアクションによって異なります。

新しいナビゲーションアクションを追加する

まず［追加］ボタンをクリックして新しい行を追加します。次に、左側のアプリケーションエクスプローラから適切なフィールドにアプリケーション要素をドラッグアンドドロップします。

アプリケーション要素の性質（具体的には、作成時にアプリケーション要素を動的にすることを選択したかどうか）によっては、適切なボタンをクリックしてアプリケーション要素にパラメーターを指定する必要があります。これにより、ビジネスオブジェクトがプロセスの一部として実行されるときに要素が識別可能になります。

次に、ドロップダウンリストからアクションを選択します。このリストは、選択したアプリケーション要素の種類によって動的に変化します。リストに表示されないアクションは、選択したアプリケーション要素に対して実行できません。

最後のフィールドは、選択されたアクションに対して何らかの入力（下のリストに記入するアイテム）があるかどうかと、それらが提供されたかどうかを示します。

各ステップの後に一時停止

遷移ステージで校正された各ナビゲーションアクションの後に、一時停止が発生するように設定できます。これは、フォームの［各ステップの後に一時停止］フィールドに式を設定することによって指定できます。設定されていない場合、ステージで生じたステップの後にランタイムエンジンは一時停止しません。

フィールド内の式は、一時停止する秒数を表す数値、または一時停止する時間を表す時間間隔のいずれかに解決される必要があります。数値が指定されている場合、小数部分はランタイムエンジンによって受け入れられます。 負の値はゼロとして扱われます。つまり、一時停止は行われません。
