Blue Prismの役割

Blue Prismの役割は、システムのユーザーの許可を設定する管理者のタスクを簡素化するように設計されています。Blue Prismでは多くのユーザーが同じタスクを実行するため、同じ許可が共有する場合があります。必要な許可セットを持つBlue Prismの役割を作成することで、管理者はその許可セットを個々のユーザーにすばやく簡単に割り当てることができます。

Blue Prismの役割を管理する

システムマネージャーにアクセスする許可を持つユーザーは、システムの役割を管理できます（つまり、役割の作成、役割の削除、役割の変更）。これを行うには、システムマネージャーの［ユーザー］ペインから［Blue Prismの役割を管理］をクリックします。ダイアログが表示されたら、左側から役割を選択して右側のチェックボックスをオンにすることで役割を変更できます。役割を削除するには、左側からその役割を選択して［削除］ボタンをクリックします。新しい役割を作成するには、［作成］ボタンをクリックし、新しい役割に名前を付けて、通常どおりに変更します。

変更内容は［OK］をクリックするまで有効になりません。［キャンセル］をクリックすると、変更内容は保存されません。影響を受けるユーザーは、関連する変更を有効にするために、ログアウトして再度ログインする必要があります。

システム管理者の役割

システム管理者の役割の許可は変更できません。また、その役割は削除できません。システムにはシステム管理者の役割を持つユーザーが常に少なくとも1人必要であり、最後のシステム管理者ユーザーが削除されるとエラーメッセージが表示されます。 システム管理者の役割を持つユーザーのみが、他のユーザーにこの役割を割り当てたり削除したりできます。

許可を設定するための役割の使用

許可を設定するための役割の使用については、「ユーザー許可」を参照してください。

シングルサインオン構成

現在のデータベースがシングルサインオン用に設定されている場合は、Blue Prismユーザーの役割とActive Directoryセキュリティグループ間のマッピングを作成する必要があります。ユーザーが適切なActive Directoryセキュリティグループのメンバーになっている場合、そのユーザーは対応する役割の許可を継承します。いずれかの役割のメンバーになっていない場合、そのユーザーはBlue Prismにサインインできません。

Blue Prismセキュリティの役割は、［役割を管理］インターフェイス内の［グループセレクター］を使用して、作成されたActive Directoryセキュリティグループにマッピングできます。Foreign Security Principalまたは未解決のSIDを持つメンバーを含むActive Directoryセキュリティグループは、クエリの問題を引き起こす可能性があるため、こうした構成はお勧めできません。特に組み込みグループや、ドメインユーザーや認証されたユーザーなどの派生メンバーシップを持つグループは、Blue Prismに関連付けられているセキュリティグループ階層内では使用しないでください。

このマッピングが確立されると、ユーザーはサインインできるようになりますが、その役割で許可されたアクションに制限されます。Active Directoryグループの変更をプロパゲートするには、ユーザーはログオフしてから再度ログオンする必要があります。
