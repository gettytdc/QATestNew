［新規リリース］ウィザード

概要

［新規リリース］ウィザードはリリースを作成するときに使用します。リリースとは、パッケージの現在の構成状態で定義されたすべてのコンポーネントを含む出力ファイルです。

ウィザードを正常に実行すると、別のBlue Prism環境にインポートされるファイルが生成され、すべてのコンポーネントと、その状態がターゲットに転送されます。

クライアントのメイン メニューから開始します。［ファイル］、［エクスポート］、［新規リリース］の順に選択するか、ショートカットキーCtrl + Rを押します。また、リリースマネージャーの［新規リリース］タスクをクリックするか、該当する［パッケージの詳細］パネルの［新規リリース］ボタンをクリックして起動することもできます。

ステージ

リリースの作成に関わるステージは次のとおりです。

パッケージ

リリースを作成するパッケージを選択します。既存のパッケージまたはこのリリース用のアドホックパッケージを選択できます。

注意：リリースマネージャーで選択したパッケージからリリースを作成している場合、このステージは表示されません。

パッケージのコンテンツ
リリースの作成元となるアドホックパッケージのコンテンツを決定します。［パッケージステージ］でアドホックパッケージが選択されている場合にのみ表示されます。
名前
リリース名を設定する
リリースノート
リリース用にリリースノートを設定する
出力ファイル
生成されたリリースの出力ファイルの宛先を設定する

パッケージ

作成するリリースの元となるパッケージはこのステージで選択します。

既存のパッケージを選択することも、リリースの作成元となるアドホックパッケージを即時作成することもできます。

リリースのコンテンツは、現在クライアントで開いているものではなく、データベースから読み込まれます。該当するコンポーネントがまだ保存されていない場合、リリースには最新バージョンが含まれていない可能性があります。

リリースマネージャーでパッケージが選択され、ウィザードがそこで起動された場合は、このステージは省略されます。

アドホックパッケージのコンテンツ

ここで、リリースを構築するアドホックパッケージのコンテンツを設定します。

リリースマネージャーで選択したパッケージからウィザードが起動した、または［パッケージ］ステージで事前定義されたパッケージが選択されているなど、ウィザードがアドホックリリースからリリースを作成するよう設定されていない場合、このステージは省略されます。

名前

リリースの名前をここで入力します。リリース名は同じパッケージ内で一意にする必要があります。

リリースには名前を付ける必要があります。1文字から最大255文字の間で入力してください。

リリースノート

各リリースには、リリース固有の指示やマニュアル的な内容を含むノートがあります。このようなノートをこのステージで入力します。

ノートは必須ではなく、入力できる文字数に制限はありません。

リリースノートは、リリースが選択されると［リリースの詳細］パネルに表示されます。

出力ファイル

リリースを保存するファイルをここで設定できます。選択したファイル名がすでに存在する場合は警告が表示されます。

出力ファイルステージには、選択したリリース名（リリースマネージャーウィザードで指定した最後のディレクトリに保存）に基づき、ファイル名が事前に入力されます。

リリースをエクスポート

ファイルを選択して［次へ>］を押すと、ウィザードにはパッケージのすべてのコンポーネントが読み込まれ、選択した出力ファイルにエクスポートされます。

エクスポート時にエラーが発生すると、そのエラーが表示されます。ユーザーは、前のステージに戻って修正できます（例：別の出力ファイルパス、または別の名前を選択します）。
