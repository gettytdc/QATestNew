Java Access Bridge

はじめに

このページには、Blue PrismのJava Access Bridge機能を使用する際のいくつかの一般的な問題と推奨される解決方法が記載されています。

要素をハイライトまたはアクセスすると対象アプリケーションがクラッシュする

要素は正常にスパイしますが、それらの要素をハイライトするなどの操作を行おうとすると、対象アプリケーションがクラッシュすることがあります。その原因は、Java Access Bridgeコンポーネントに欠点があり、1つの画面で多数（200個以上）の要素が表示されるときに問題が生じることが考えられます。 クラッシュしないようにするには、アプリケーションモデラーで 'descendtree' 対象アプリケーションオプションを有効にします。このモードを有効にすると、速度が大幅に低下するため、必要な場合にのみ使用してください。

JABのバージョン

インストールされているJREには、最新のJABバージョンを使用することをお勧めします。JRE7u6以降、JAB 2.0.3はJavaインストーラーにバンドルされています。Jre7u60以降、JAB 2.0.4はバンドルされています。

JREのバージョンJABのバージョン
JRE5以下2.0.1または2.0.0
JRE5以降2.0.2
JRE6以降2.0.2
JRE7以降2.0.4

Blueprismは、JREのバージョンにJABがバンドルされていないJABのさまざまなバージョンをインストールできるBPJabInstaller-1.0.2.exeを提供できます。

JABがJavaインストーラーにバンドルされている場合は、JABを有効にする必要があります。次のコマンドラインを使用して実行できます。コマンドプロンプトを開き、java binディレクトリ（例：C:\Program Files\Java\Jre7\bin）に変更します。次に、以下のコマンドを実行します。

jabswitch.exe /enable

JABはWindowsの［簡単操作］コントロールパネル（Windows + Uを押してアクティベート可能）で有効にすることもできます。［簡単操作］コントロールパネルには、［Java Access Bridge］チェックボックスがあります。

スパイツールが要素を認識しない

スパイツールが要素の上に移動したときに要素を認識していないようであれば、まず、a）Java Access Bridgeのインストール後、再起動していない、b）Java Access Bridgeの間違ったバージョンを使用している、といったよくある原因について確認してください。

テキストボックスに書き込もうとするとエラーメッセージが表示される

「setTextContentsの呼び出しに失敗しました」などのエラーメッセージ。この方法は、Java Runtime Environmentのバージョン1.3.xに対しては機能しないことがわかっています。Java Runtime EnvironmentはJRE 1.3で実行されているアプリケーションでJavaテキストボックスへの書き込みを試行していることを示しています。 Java Access Bridgeコネクターに、そのJREバージョンに対応する実装がないため、この操作は行えません。

いずれにせよ回避策はありますが、ユーザー側である程度の工夫が必要となる場合があります。よくある問題への推奨される回避策は以下のとおりです。

よくある回避策 - Java Runtime Environment（JRE）をアップグレードする
ほとんどの場合、アップグレードしても対象アプリケーションの機能はまったく変わりませんが、その前に潜在的な問題によるリスクを考慮しておく必要があります。Blue Prismは、1.4.0以降のバージョンでは問題なく機能します。
書き込み問題を回避するには
要素へのデータの書き込みに問題がある場合は、代わりにアプリケーションにキーストロークを送信できます。これには、目的の要素を含むウィンドウをアクティブにし、要素がキーボード入力を受信する準備ができた状態にして、［ナビゲート］ステージを使用してキーストロークを送信することが含まれます。要素内の既存のデータを消去するには、キー ストロークに［Backspace］キーや［Delete］キーを含める必要があります。
要素へのアクセスが非常に遅い

Java Access Bridgeには、いくつかの具体的で利用可能なパフォーマンスのヒントがあります。
