应用程序建模器

应用程序参数向导

简介

在 Object Studio 中创建业务对象时，第一个任务通常是定义目标应用程序。这包括向 Blue Prism 提供有关目标应用程序的信息，以及在应用程序建模器中构建应用程序定义。

或者，如果目标应用程序已经在另一个可共享业务对象中定义，则可以通过选择该对象作为父对象来使用该模型。

访问向导

如果之前未运行该向导，则在您第一次尝试打开应用程序建模器时会出现该向导。这是通过按 Object Studio 工具栏上的相应按钮来实现的。也可以通过以下方法再次启动该向导：打开应用程序建模器，选择应用程序定义树形结构的根节点，然后在树对面的窗格中点击相应的按钮。

完成向导

选择应用程序类型

第一个任务是选择应用程序类型。应用程序类型的名称应不言自明，而且显而易见的选择应能自我防错。如果有疑问，则应选择 Windows 应用程序选项。

请注意，某些应用程序类型受许可限制的约束。这可能会限制您的选择，阻止您选择最适合的应用程序类型。如果您发现自己处于这种情况，请联系 Blue Prism 寻求建议。

提供更多信息

选择应用程序类型后，通常要回答一系列问题。后续问题通常取决于为前面的问题提供的回答，特别是取决于应用程序类型。总体来讲，这些信息称为参数集，它们详细描述目标应用程序的性质及其运行时环境。有关各个参数的信息，以及有效值范围的描述，请参阅参数帮助。
