Process Studio

Process Studio 在 Blue Prism 软件包中起着关键的作用—它使设计人员能够通过使用带有拖放流程组件的图形绘制界面来创建、编辑和测试流程。

流程区域（白色主区域）是屏幕的一个区域，在该区域内，设计人员可以使用工具箱中的流程图组件来设计和测试流程。要绘制流程，请从工具箱中选择一个绘图对象，然后点击画布上要显示该对象的区域。或者，将对象从工具箱拖放到画布上。

当鼠标指针悬停在画布上时，指针会改变以显示当前选定的对象。设计人员可以使用屏幕顶部的“查看”>“动态光标”菜单选项禁用此行为。

提供的网格使流程内的对象更容易对齐，并且“对齐网格”选项能让放置的每个对象与最近的网格线对齐。设计人员可以通过“查看”菜单启用或禁用这两个选项。

设计人员可以使用“查看”>“缩放”菜单缩放到不同的精度，还可以通过“编辑”菜单实现“撤销”和“重做”功能。

以可视方式设计和检查完流程图之后，设计人员最好在调试模式下运行流程，消除流程中的问题，然后发布流程，以供在控制室中使用。

您可以在流程图帮助页面上了解有关设计流程的更多信息。

环节日志记录和警告

在流程执行期间，用户可以选择将每个环节的详细信息记录在会话日志中；此操作是通过在相关环节属性屏幕上使用“环节日志记录”选项来实现的。以下选项可供选择：

  • 已启用—环节始终记录在会话日志中
  • 已禁用—环节从未记录在会话日志中
  • 仅错误—仅在遇到错误时才将环节记录在会话日志中

对于未禁用日志记录的操作环节，还可以通过确保未选中“不记录此环节的相关参数”复选框来记录任何输入/输出参数的值。（请注意，密码数据项始终被遮蔽）。

系统默认在为流程创建的所有新环节以及为对象创建的所有新异常环节上都启用日志记录。可以通过“编辑”菜单为所有（或选定）环节更改日志记录选项。

如有必要，还可以在单个资源级别上覆盖环节日志记录。请参阅资源管理。

有关查看生成的日志的信息，请参阅会话日志。

默认情况下，如果在一个流程中用于执行某一环节的时间超过了系统设置中配置的阈值，则会话状态将更改为警告并在控制室中突出显示。但是，可根据需要在单个环节级别上覆盖警告阈值，以适应流程执行期间的预期延迟。

将警告阈值设置为 0 分钟将禁用环节的此功能。
