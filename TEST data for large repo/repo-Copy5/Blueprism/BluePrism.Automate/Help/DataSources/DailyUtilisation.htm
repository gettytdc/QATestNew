Blue Prism

Daily Utilisation

BPDS_DailyUtilisation

Returns overall resource utilisation over the given period, i.e. the amount of time that the resources have spent actively running sessions. Data is only returned for completed days (i.e. sessions created after the the MI statistics were last compiled are not included).

Source: Blue Prism Database – Live tables and MI Repository (requires compiling of MI statistics to be enabled)

Sample Chart

Parameters

Parameter Name Data Type Description Validation
@BPResourceName Text The name of an individual resource to be queried (if not specified all resources will be queried) None.
@NumberOfDays Integer Reporting period in days (if not specified the number of days is 7) Max. 31 days
@DisplayUnits Text

Possible values are:

  • Second – values are returned in units of seconds
  • Minute – values are returned in units of minutes
  • Hour – values are returned in units of hours

  • Percentage – values are returned as a percentage of the total processing time available (e.g. If processing has been carried out by 5 resources on a given day then total processing time would be 5 x 24 (resources x hours) = 120 processing hours. Therefore a result of 50% would suggest that 60 hours of processing time were achieved in the given time frame)

If not specified the utilisation values are returned in minutes

 Second, Minute, Hour or Percentage
@MaxResourceHours Integer The maximum number of processing hours for a resource. This value will be used to calculate utilisation where displayed as a percentage, see @DisplayUnits (if not specified then 24 hours is assumed). Max. 24 hours
