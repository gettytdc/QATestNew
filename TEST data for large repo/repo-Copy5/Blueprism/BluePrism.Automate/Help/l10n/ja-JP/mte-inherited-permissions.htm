継承される許可

アクセス権がグループに設定されると、それらはそのグループ内のすべてのプロセス、オブジェクト、リソースに適用されます。これには、すべての子グループとそのコンテンツも含まれます。これはグループやアイテムを別のグループに移動するときにも該当します。すべてのアイテムは移動先のグループの許可を継承し、すでに適用されている許可はすべて上書きされます。グループまたはアイテムが移動され、許可が変更されるたびに、メッセージボックスが表示され、移動によって影響が出ることが通知されます。

グループに制限がない場合、子グループのアクセス権を必要に応じて設定できます。制限付きのグループの場合、子グループのアクセス権は表示できますが、編集はできません。

グループを移動する

グループを移動するには、次の許可が必要です。

  • グループを移動するには、グループを編集する許可が必要です。
  • 移動に関与するあらゆる制限付きグループには、アクセス権を管理する許可とグループを編集する許可が必要になります。

グループには直接的または間接的に制限を与えることができます。

  • 直接 - 許可は、そのグループに個別に適用されるアクセス権によって決定されます。
  • 間接 - 許可は、親グループに適用されるアクセス権によって決定されます。
