MI 报告

仪表盘信息基于存储在 Blue Prism 数据库中的信息。每个仪表盘磁贴的源数据都是单独的 SQL Server 存储过程（保存的查询），它从数据库模式的特殊管理信息表中检索数据。有关磁贴数据源的更多信息，请参阅数据源。

在“报告”屏幕中启用使用率和生产力统计信息时，将从数据中填充统称为管理信息 (MI) 的数据的源表。

数据库中有三种存储 MI 数据的表：

  • 每日原始数据—包含当天的使用情况统计信息
  • 每日合并数据—包含针对已定义的每日保留期的合并统计信息
  • 每月合并数据—包含针对已定义的每月保留期的合并统计数据

每日原始数据表通过启用使用率和生产力统计信息的集合来填充。要启用统计信息记录：

  • 点击“系统 - 报告”。这将显示“MI 报告”设置。
  • 要启用统计信息收集，请选中复选框“收集使用率和生产力统计信息”。

MI 数据刷新设置

启用统计信息收集后，有两个选项可用于控制刷新统计信息数据的任务。刷新任务会从每日原始数据表中移动数据，进行聚合，然后将其传输到每日和每月合并数据表中。

  • 每天在指定时间自动刷新统计信息
    • 使用 Blue Prism 服务器服务运行刷新任务
    • 将数据从原始表移动到合并表中
    • 删除任何超出保留设置的数据
  • 使用外部工具刷新统计信息

    • 如果 Blue Prism 没有自动刷新 MI 统计信息，则可以将存储过程“usp_RefreshMI”配置为每晚执行。实现这一目标的选项包括：

    • 使用 SQL Server 代理设置计划任务
    • 使用 Windows 计划任务通过 SQLCMD 实用程序执行存储过程
    • 设计和安排使用“数据—SQL Server”可视业务对象的 Blue Prism 流程

保留期设置

每日和每月数据集可以在所定义的期间内保留。可以使用 MI 报告设置页面中的调节控件设置具体的天数和月数。 需要使用“应用”按钮来应用对这些设置的任何更改。

天数和月数决定了每日和每月合并数据表中保留的数据量。刷新数据时，将自动从合并表中删除超过定义的保留期设置的数据（请参阅 MI 数据刷新设置）。触发每日刷新作业时，每日原始数据表中的数据将传输到每日合并数据表，然后清空原始表。

系统报告

系统管理员可以使用系统报告收集可用于分析和调试安装的有用信息。

要运行报告，只需从列表中选择所需的报告，然后点击“生成”按钮。您将看到一个“另存为”对话框，您应在其中选择文件名并点击“保存”。
