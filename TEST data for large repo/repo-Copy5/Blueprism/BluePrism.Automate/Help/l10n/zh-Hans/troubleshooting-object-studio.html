故障排除

Object Studio

简介

此页面提供使用 Blue Prism 的 Object Studio 功能的一些常见问题和建议的解决方法

在探测应用程序元素之后，Blue Prism 没有立即为我突出显示它

此问题是由于对目标应用程序中的应用程序元素的识别方式存在误解。探测工具收集有关目标元素的信息（属性列表）并将其显示在列表中。

默认情况下，一些更有用的属性将标记为必需，而其他属性则不会。该选择不代表已知适合于识别元素的确定属性集；而只是猜测。用户可以查看列表，并为相关元素选择最佳属性集。

请参阅选择必需的属性，了解详细信息。

上次测试我的业务对象时，它能正常工作，但重新加载我的对象后，它似乎不再有效

每次重新加载目标应用程序时，其元素的功能都可能会发生重大和细微变化。例如，按钮上的文本可能会从“更新”更改为“提交”（重大变化）或，元素的创建顺序（称为序数）可能发生变化（细微变化）。

作为业务对象设计者，您需要注意标记为必需的属性，只有这些属性在每次运行应用程序时保持不变。如果每次的序数不同，则最好不要标记为必需。

在面对其元素不断发生变化的应用程序时，您可以采用各种策略：

  • 属性更改可预测

    例如，按钮上的文本可以根据某个其他变量从一个值更改为另一个值。在这种情况下，您可以使文本动态显示来更新元素。

  • 属性更改不可预测但处于有限范围内

    例如，元素的序数可能有所不同，但总是在 2-4 的范围内。在这种情况下，您可以决定复制并粘贴元素以复制它，将每个副本中的序数更改为新值。然后，在业务对象中，您可以使用带有 CheckExists 查询的等待环节将流分支到每种情况下的不同点。

    请注意，通过使序数动态显示可以实现类似效果。

请参阅选择必需的属性，了解详细信息。

我的元素在设计时被成功识别，但在运行时不能被成功识别

此问题可能是因为您的某些属性在应用程序运行期间发生了变化。

例如，如果您的元素是要向其发送文本的文本字段，则不应将其文本用作必需属性，因为其文本将在您写入后发生变化。

此问题的表现是业务对象在写入文本字段之前的前几分钟工作，此后无法工作。

我在检测元素在被禁用或启用时遇到问题

这可能是因为 Microsoft Active Accessibility 中使用的术语混淆。使用不可用属性而不是已启用属性。

在调试期间，Object Studio 始终将自己置于焦点之中。这是需要将目标应用程序设为焦点时遇到的一个问题

您可以通过以下方式来控制 Object Studio 的行为：

  • 使用任务托盘中的调试按钮（查看屏幕右下方的 Prism 图标；使用上下文菜单选择选项）。此方法不会影响 Object Studio 的焦点。
我的业务对象在一台资源 PC 上正常运行，但在另一台资源 PC 上不能正常运行

这个问题有很多可能的原因。实质上，通常是因为目标应用程序在第二个资源 pc 上的外观或行为不同。

这种差异可能是因为每台目标计算机上使用的软件版本（您应该检查所有资源 PC 上是否安装了相同的版本）；或者使用的主题（例如 Windows XP 主题与 Windows 经典主题）—使用的主题通常会影响应用程序元素的类名；或硬件差异—例如影响应用程序布局的不同屏幕分辨率。

要解决此问题，您需要在所有资源 PC 上使目标环境尽可能统一，或仔细选择您的元素属性，以便可以使用相同的属性在每个不同的环境中进行识别。

请注意，后一种解决方案将需要进行大量测试，并将使开发流程更加复杂和困难。

在调试期间，Object Studio 始终将自己置于焦点之中。这是需要将目标应用程序设为焦点时遇到的一个问题

您可以通过以下方式来控制 Object Studio 的行为：

  • 使用任务托盘中的调试按钮（查看屏幕右下方的 Prism 图标；使用上下文菜单选择选项）。此方法不会影响 Object Studio 的焦点。
我的应用程序在一个窗口下打开所有文档，这意味着我无法启动自己的新实例或探测它

某些应用程序在启动时会收集必需信息（例如，打开新文档），将此信息传递到现有实例，然后退出。一个这样的应用程序是 Microsoft Word。

此问题的解决方案是附加到应用程序的现有实例。附加可以在任何时候完成—之前或之后，但是它要求应用程序已经启动。您必须实现一些逻辑来检测应用程序是否已在运行，并在每种情况下适当地启动/附加。

我无法与 Internet Explorer 文件下载对话框进行交互

此问题是由于 Internet Explorer 在本地配置的方式造成的。您可以联系 Blue Prism 来讨论您的 Internet Explorer 配置或使用以下变通方法：使用“激活”操作将窗口置于前台，等待五秒钟，然后在下载对话框上点击所需按钮。

Javascript 无法正常工作（访问被拒绝）

在网页中调用 Javascript 时，您可能会遇到不同的结果，具体取决于您的 Internet Explorer 版本和配置。特别是根据安全设置的不同，您可能会在尝试调用 Javascript 方法时收到“访问被拒绝”错误。您的代理服务器设置可能会影响 Internet Explorer 实施的安全限制。在可能的情况下，您应取消选中“自动检测设置”复选框，因为这会导致实施更严格的规则。

Active Accessibility 性能低下

尝试在网页或复杂应用程序中查找唯一的 Active Accessibility 元素时，性能可能会很低。这是因为必须测试大量的元素才能找到唯一匹配。为了加速性能，可以为 MatchIndex 属性设置一个值。其方法是指定 Blue Prism 在达到指定的匹配数后停止搜索。虽然这确实提高了性能，但您应该知道，通常，如果没有此属性集，Blue Prism 将搜索整个对象模型。Blue Prism 在执行此操作时确保匹配是唯一的非常重要。如果您确定匹配是唯一的，则可以使用此属性。

在使用侵入式技术时，Blue Prism 无法在其附加的应用程序中找到元素

为应用程序模型启用“侵入式技术”时，附加到应用程序可能会使一些控件无法访问或不在 Blue Prism 中的模型内显示。

发生这种情况的原因是，在启用侵入式技术的情况下，Blue Prism 会侦听在附加应用程序中创建的控件。如果应用程序在 Blue Prism 附加到它之前创建其控件，则无法知道这些控件是否存在，因此也无法引用它们。

为了使用这些控件，应该在创建控件之前附加应用程序，或者应该使用 Blue Prism 来启动相关应用程序，确保它可以完全访问其所有控件。
