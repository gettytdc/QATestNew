集合环节

集合环节

有关集合使用的概述，请参阅集合帮助页面；此页面针对完成集合环节的属性窗体给出了实用性的指南。“集合属性”对话框具有多个要填写的字段。

名称

您输入的名称可识别集合，且该名称将在流程图上显示。

描述

附加的描述字段允许输入更长的叙述。

字段

使用以下两种方法中的一种向集合添加字段。

使用业务对象中的数据
要使用业务对象添加字段，请点击“导入字段”按钮。有关此选项的更多帮助，请参阅此页面的“导入字段”部分。
手动
要手动添加字段，请根据所需字段数点击“添加字段”按钮。每次按下按钮后，将向集合中添加一个字段。接下来，使用下拉菜单为每个字段选择一个数据类型

将字段添加到嵌套集合中

一个集合可以包含其他集合，添加一个字段并选择数据类型“集合”，以便手动定义这些“嵌套”集合中的字段。“字段”列下会显示一个标签，指示未定义新集合的字段。选择该行，然后点击链接以定义此集合的字段。

在定义嵌套集合的字段时，将显示“痕迹导航跟踪”，指示当前正在编辑的集合定义。要返回到父集合，可以在痕迹导航中点击其名称。

痕迹导航跟踪中的名是集合本身的名。点击此操作将返回到基本环节配置，其中也可以修改环节的其他属性。

单行

单行集合只有一行——您可以通过勾选“字段”选项卡中的复选框，将集合（在环节中定义的主集合或嵌套集合）设置为单行。

注意： 如果已经为集合添加了行，只有选中此复选框仅保留一个之后，这些行才会被删除。相反，如果集合当前没有行，则将添加一行。

添加初始值

您可以选择为集合提供一组初始值。要添加初始值，请点击“初始值”选项卡。要向集合初始值添加新行，请点击“添加行”按钮。每次点击该按钮都会新创建一个行。添加一行后，可以通过点击行中的单元格来设置值。

在值中填充嵌套集合

如果集合包含其他集合，则可以通过选择要编辑的值中的行并点击表示集合的超链接来设置它们的值。

如果集合没有数据，则链接将显示“无数据”；如果集合是空的，即，该集合是一个没有行的集合，则会显示“空”。如果该集合有任何行，则会显示“第 m 行，共 n 行”，其中“m”表示集合的当前行，“n”表示总行数。

选择该行后，可以使用标记为“清除”的按钮。此按钮将删除集合，即，将其重置为“无数据”。

当正在编辑嵌套集合的值时，系统会出现“痕迹导航跟踪”，指示正在修改值的级别。要返回到父集合，可以在痕迹导航中点击其名称。

痕迹导航跟踪中的名是集合本身的名。点击此处将返回到基本环节配置，其中还可以修改集合的结构及其嵌套集合等环节的其他属性。

当对话框当前导航嵌套集合的值（初始值或当前值）时，可以查看“字段”选项卡，但无法通过该选项卡编辑这些字段。要编辑字段，必须通过点击痕迹导航路径中的名来导航到环节本身。

从操作导入字段

如果操作接受作为输入的集合，或者将集合作为输出发出，则可以通过点击“环节属性”对话框中的“导入字段”按钮将该操作中定义的字段导入到集合环节。

激活后，系统将打开一个对话框，其中列出了所有可用的业务对象以及参数中具有已定义集合的操作。

选择对象时，系统将列出其适用的操作。选择操作时，系统将显示具有已定义集合的参数。

选择参数时，系统将显示该参数中定义的字段的预览。按“确定”会导致字段定义被传输到当前打开的集合定义。

请注意，集合定义可以导入到任何嵌套集合中，而不仅仅是最顶层的集合，例如，可以拥有一个带有 Web 服务名称和 URL，和一个带有从 Web 服务业务对象导入定义的“输入”集合字段的“Web 服务”集合，以及一个从同一个 Web 服务业务对象的输出中导入定义的单独输出集合。
