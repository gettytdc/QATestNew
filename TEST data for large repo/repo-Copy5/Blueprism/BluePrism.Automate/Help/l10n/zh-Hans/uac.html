Blue Prism 配置

Windows Vista/7 用户帐户控制

概述

Windows Vista 向 Windows 环境引入了“用户帐户控制”—在这种机制下，如果要进行影响所有用户的系统更改，则可以通过在进行此类更改时请求提升权限来对其控制。

这如何影响 Blue Prism 软件

从 Blue Prism 的角度来看，这会影响以下任何配置更改：—

  • 客户端和资源 PC 连接的数据库/服务器连接列表。
  • 在计算机上启动 Blue Prism 客户端应用程序时是否运行资源计算机
  • 在计算机上注册的外部业务对象
  • 在计算机上定义的 Blue Prism 服务器配置

如果 Blue Prism 以管理员身份运行（即程序配置为始终以“以管理员身份运行”），则它将保存配置，而不会再向用户发出提示。

如果应用程序在正常权限下运行，则用户将在第一次需要保存计算机配置时收到提升提示。如果接受，将运行名为 AutomateConfig 的单独程序，并且只要 Blue Prism 客户端正在运行，该程序就将继续运行。

您可以通过查看通知区域（屏幕右下角）来判断 AutomateConfig 何时运行。将有一个 Blue Prism 图标，上面覆盖一个小齿轮。

如果程序保持打开的时间超过其应当打开的时间，可以通过右键点击图标并选择“退出”来关闭程序
