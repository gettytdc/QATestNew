帮助

自动保存

在 Process Studio 中操作时，Blue Prism 可以自动备份您所执行的操作，以防出现系统崩溃、电源故障和其他无法预料的故障。此类备份不会覆盖您的现有流程，而是会记录到 Blue Prism 数据库中的某个独立位置，以便您选择是要还原为未经更改的流程，还是要用备份中所含的版本来覆盖现有流程。

自定义备份间隔

默认的备份间隔可在系统管理器中进行设置。但是，每位用户都可以通过 Process Studio 中的文件菜单来调整备份间隔，以覆盖默认设置。Blue Prism 会记住每位用户的具体设置。这样，即使用户在不同的计算机上访问 Process Studio，Blue Prism 也能按照所设置的间隔进行备份。

备份

备份将在后台进行，无需用户操心。每次进行备份时，Blue Prism 都会在 Process Studio 的状态栏上单独显示相关消息，以通知用户备份正在进行。

从备份恢复

如果出现系统故障，Blue Prism 将不允许针对受影响的任何流程进行进一步编辑。要继续处理相关流程，您必须采取以下两种措施之一：必须恢复流程的初始版本；或者，必须恢复最新的备份。如果在恢复了流程的某个版本后，您发现最好改为恢复另一个版本，那您也能轻松地做到―只需像恢复当前版本一样，恢复所需的版本即可。在提交某个特定版本之前，用户可以随意地多次改用不同的版本。当准备好提交某个特定版本时，用户应该点击 Process Studio 中的“保存”按钮。此时，所有的现有备份都将被清除，而且此操作将无法撤销。

恢复要通过系统管理器来进行。请参阅备份恢复向导的帮助页面，以进一步了解相关详情。
