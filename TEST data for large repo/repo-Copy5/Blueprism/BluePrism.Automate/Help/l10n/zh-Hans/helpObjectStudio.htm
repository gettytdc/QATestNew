Object Studio

概述

Object Studio 是版本 3.0 中引入的一个新功能，旨在提供一个与 Process Studio 中一样的流程图界面，以简化业务对象的创建过程。这类业务对象被称为可视业务对象。

通常，可视业务对象会被用于与未提供 API 的传统 Windows 应用程序进行交互。对于提供了 API 的应用程序，则更适合使用 COM 业务对象使应用程序实现自动化。但这一建议不适用于大型机应用程序：由于这类应用程序使用广泛，所以 Blue Prism 提供了专门的功能来实现与这些 API 的快速集成。

Object Studio 的使用

与 Process Studio 中一样，相关环节会被置于流程图中的各个位置，然后通过决策和选择环节来控制流程的执行方向。

页面

业务流程会从主页面上的开始环节开始执行，然后沿着环节之间的连线继续执行。每个页面都是整个自然流中的一个逻辑子部分。页面有两个主要用途：a) 重复使用经常重复的环节组；以及 b) 将整个流程图整理得井然有序。因此，任何流程实际上都能在单个页面上实现等效呈现。

相反，业务对象中的公开页面则无法做到部分呈现；每个页面本身就是一个实体，这类实体被称为功能，拥有自己的起点。 从这层意义上来说，业务对象并没有唯一的起点：只是包含很多功能，而每个功能都有自己唯一的起点。

当流程使用业务对象时，前者会直接调用相关的功能（如选定的页面）；整个执行过程都不会涉及主页面。

发布页面

如上所述，每个页面都会向业务对象呈现一个唯一的入口点。默认情况下，所有这些入口点对于 Blue Prism 流程都是不可见的。只有在页面发布后，它们才会变为可见状态。这可通过页面标题中的上下文菜单选项来实现。完成发布后，您便可照常通过操作环节来调用相应的功能。

如果某些页面对您的业务对象而言非常有用，但您不想公开这些页面，那么您可以让这些页面保持未发布状态，以使其成为“私有”页面。同样地，这也是一种非常有用的策略，可确保新功能在准备就绪并经过全面测试后才可供使用。

新的环节类型

Object Studio 引入了下面几种新的环节类型：读取、写入、导航、代码和等待。这些环节类型仅适用于 Object Studio，它们不适用于 Process Studio。这些新环节用于控制与目标应用程序的交互：读取环节和写入环节控制数据的获取和输入；导航环节可用于打开菜单、点击按钮等；等待环节用于确保目标应用程序已准备好进入下一环节（即已完成加载）；代码环节允许高级用户使用 COM 对象。

应用程序建模器

应用程序建模器是 Object Studio 中的一个关键功能。要为任何应用程序构建业务对象，您首先要做的就是将目标应用程序的各个元素告知 Blue Prism。这涉及：创建友好的名称（如“‘确定’按钮”）；然后，告知 Blue Prism 可以在何处通过某个点找到这个元素、应该在何处使用鼠标进行点击操作。请重复这个过程，直至完成目标应用程序中所有元素的定义。其中包括您想要从中读取数据的所有文本框、列表视图、组合框等，还包括您需要使用的所有菜单、按钮和选项卡控件。

在定义好目标应用程序的元素后，您便可在上文所述的各个新环节中使用这些元素。应用程序建模器中的应用程序元素树视图会出现在新环节的属性窗体中。只需将元素从树中拖放到属性窗体的新行内，即可使用这些元素。例如，要从应用程序的某个文本框中读取一些文本，您可以将文本框元素拖到读取环节的新行中，然后再将数据项拖到同一行中。当执行该环节时，文本会从目标应用程序中读出并存储到所选的数据项中。
