应用程序属性—UI Automation 模式

简介

以下是 Windows 探测工具在 UI Automation 模式中返回的属性列表。有关由其他模式返回的属性列表，请参阅：—

  • Win32 模式
  • Active Accessibility 模式
  • Java 模式
  • HTML 模式

本页描述的属性大致对应 MSDN 的“自动化元素信息”页面中描述的 UIAutomation 元素属性。

应用程序属性

匹配索引
可用于从若干匹配元素中标示出要识别的元素的通用属性。
UIA 快捷键
由目标应用程序用于指示可以调用元素的组合键，例如：可以指定 Ctrl + N 来指示您可以通过按住 Ctrl 键并按 N 来调用元素。
UIA 访问键
菜单、菜单项或控件标签的文本中的字符，用于激活附加功能。例如，字母“O”通常用于从“文件”菜单中调用“打开文件”对话框。
UIA 自动化 ID
在可用时，无论使用何种本地语言，元素的自动化 ID 在应用程序的任何实例中都应该是相同的。
UIA 类名称
由控件开发人员为元素指定的类名。
UIA 控件类型
控件类型的描述，如“按钮”。无论使用何种本地语言，元素的控件类型在应用程序的任何实例中都应该是相同的。
UIA 已启用
指示元素是已启用 (True) 还是未启用 (False)。
UIA 具有键盘焦点
如果元素当前具有键盘焦点，则为 True；如果不是，则为 False。
UIA 高度
以像素为单位的元素高度
UIA 帮助文本
与元素关联的帮助文本
UIA 项状态
元素中项状态的描述。
UIA 项类型
项类型的本地化描述。
UIA 标记方
包含此元素的文本标签的元素的自动化 ID。
UIA 本地化控件类型
控件类型的本地化描述，如“按钮”。应用程序实例的本地化控件类型将以本地语言显示。
UIA 名称
元素的名称。界面开发人员经常使用它来保存元素的固有值。
UIA 屏幕外
如果元素完全滚动或折叠到不可见，则为 True。如果元素有可点击点，可以导致它获得焦点，那么即使元素的一部分不在屏幕上，也会认为它在屏幕上。
UIA 方向
只与某些元素类型相关。此属性指示元素的方向—无、水平或垂直。
UIA 密码
如果元素包含受保护的内容，则为 True，否则为 False。
UIA 流程 ID
从中识别此元素的流程的流程 ID。
UIA 必需
如果指示必须在窗体上填写此元素，则为 True；如果无此指示，则为 False。
UIA 顶级窗口 ID
承载此元素的顶级窗口的自动化 ID。
UIA 宽度
以像素为单位的元素宽度
UIA X
元素相对于屏幕原点的 X 坐标。
UIA Y
元素相对于屏幕原点的 Y 坐标。
