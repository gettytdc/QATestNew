Process Search

The Process Search at the top of the Process Studio design window allows the user to easily find a stage within the current Process that contains an entered phrase. The search will look for the phrase in both the Name and Description of every stage in the Process.

The Search tool applies equally to Visual Business Objects in Object Studio.

Appearance and Behaviour

The search bar is located in the toolbar towards the top of the screen. It appears as illustrated:

If the entered search phrase is found the stage it is in will be highlighted and selected in the flow diagram window. If the phrase is not found then a suitable message will be displayed in the status bar. The user may continue to find subsequent occurrences of the search phrase within other stages by repeated presses of the enter key.

Advanced Search

If the user presses the Advanced Search option the Advanced Search window will be displayed. The Advanced Search window appears as illustrated:

As with the Search Bar, if the entered search phrase is found within the name or description of a stage, the stage will be highlighted and selected in the process design window. If the phrase is not found an error message will be displayed in the status bar. The user may continue to find subsequent occurrences of the search phrase within other stages by simply re-pressing the enter key or pressing the Find Next button.

The Advanced Search window has the following additional functionality:

Match Case

If the Match Case box is checked the search will only find stages containing the same phrase with the same upper and lower case characters. For example, if the search phrase entered was "tEst", a stage which only contained the name "test" would not be found, due to the mismatch of upper- and lower-case letters.

Match Whole Word

If the Match Whole Word box is checked the search will only find stages if they contain a whole word that matches the entered search phrase. For example, if the search phrase entered was 'test' a stage which only contained the name 'test1' would not be found.

Search Up

If the Search Up box is checked this search function will look for the entered search phrase upwards from the current stage (or from the last stage if no stage is active) instead of downwards.

Details

The Details button toggles the display of a treeview and listview boxes as shown in the above picture. The treeview will highlight every stage which contained the entered search phrase, and the listview will highlight the most recent occurrence of the search phrase to be found. To change the appearance of the treeview and listview the user may resize the Advanced Search form, adjust the border between the treeview and listview, or rearrange the order of the columns in the listview by dragging them to a new position.

Regular Expressions

When regular expressions are enabled (via the checkbox), Blue Prism will interpret the text entered into the search box as a regular expression. Each matching stage will be highlighted in turn, with repeated presses of the enter key.

See the wildcards and regular expressions page for more information.

Element Searches

Within advanced search a checkbox exists for searching for stages which use an application element. For example if you wanted to find references to application elements containing the word "button" you can check the Search Element checkbox and all stages that use elements containing the word "button" will be found.

The familiar options such as match case are still available in the same way. For example if the above search was performed using the match case option, then it would match the element "OK button" but not the element "Cancel Button" (due to the upper-case versus lower-case "b" on "button").
