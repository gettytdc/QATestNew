Studio Permissions Summary

Overview

The following permissions can be used to control what particular users can do with Objects and Processes within Studio (and other parts of the application). Refer to User Permissions for more information about how permissions are assigned to users through User Roles.

PermissionEntitlement
Create Users can create items within Studio, including using the Save As option in the editor. Edit, Execute and View permissions are also assumed
Edit Users can edit items within Studio and publish/unpublish Processes via AutomateC. Execute and View permissions are also assumed
Execute Users can only execute items (i.e. they canâ€™t step into the item while debugging their own Object or Process). Execute does not automatically grant View, and users needing to open an object/process would require View or Edit permissions
View Users have a read-only view of items. E.g. right-click from control room or using the Object/Process comparison tool. If the user also has the Execute permission, then they would be able to step into the item while debugging their own Object or Process.
Delete Users can delete items from within Studio
Export Users can export items from the File menu, Object/process editor, AutomateC or the Release tab (assuming they have access to it)
Import Users can import items from the File menu, AutomateC or the Release tab (assuming they have access to it)
Edit Groups Users can create/rename/move/delete groups within the Studio tree
