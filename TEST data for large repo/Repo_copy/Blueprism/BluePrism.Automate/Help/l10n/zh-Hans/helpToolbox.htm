Process Studio—工具箱

Process Studio 中提供以下绘图工具：

指针

允许对象在画布上突出显示和移动

缩放

实现缩放功能，左键点击流程放大，右键点击缩小。

操作

对流程执行操作。右键点击并选择属性，可定义要使用的操作库以及该库中特别需要的功能，例如 Microsoft Word 库、打印文档功能。

决策

决策环节用于确定给定流程中的流程流。决策只能有 True / False 答案，并且需要引入一个向内链接，引入两个向外链接。要指定决策的构成，请转到决策环节的属性表，并指定比较方法和两个变量或要比较的常量，例如[Arrears] > 500。

链接

链接将流程图中的各个环节连接在一起，并可使用切换选项快速反转。大多数对象可以有多个向内链接，但只能有一个向外链接，但数据（没有链接功能）、锚定（可以有多个向内链接）和决策（有两个向外链接）除外。

计算

在将结果放入预定数据框之前，计算将操纵当前流程中的数据。例如，您可能希望将两个数字相加，连接两个或多个文本，或者舍入余额。请参阅计算帮助，了解详细信息。

锚定

锚定环节是一个可视化环节，允许将链接放置在流程周围的直角处，从而防止流程难以在页面上布局。

数据

数据框包含给定操作环节的输出，并用作信息的容器，可在流程图的稍后决策和操作中使用。

集合

集合是一种特殊类型的数据框，可以同时保存多个数据项。集合只能填充具有类型集合输出的操作。

循环开始

循环开始环节标记循环的开始。循环通常用于迭代集合。也就是说，循环中包含的许多操作依次对集合中的每个项执行。

循环结束

循环结束环节标记循环的结束。在下一次循环运行开始之前，运行的流程必须达到这个位置。

备注

备注包含有关流程图或元素的备忘录信息，并显示存储在叙述中而不是名称中的文本。默认情况下，首次创建流程时，会在流程图上引入单个备注并指定名称和描述。

开始

所有流程图必须具有第一个链接流经的开始点。

结束环节

所有流程图必须至少有一个最终链接在此终止的结束点。
