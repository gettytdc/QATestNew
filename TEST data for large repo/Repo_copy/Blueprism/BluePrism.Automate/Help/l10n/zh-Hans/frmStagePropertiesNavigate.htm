环节类型

导航环节

您可以通过导航环节在目标应用程序中执行导航步骤（或操作）。相关示例包括点击按钮、打开菜单或对话框、关闭窗口、发送击键等。

属性窗体

导航属性表单中共有上下两个列表。第一个列表中列有要执行的导航操作。这些操作将从上至下依次执行。例如，第一个操作可能是打开“文件”菜单，第二个操作可能是点击“保存”。这里的操作排列顺序非常重要，所以您应该小心处理，以便按照正确的顺序来添加导航操作。

每个导航操作都由应用程序元素和相应的操作组成。应用程序元素是操作所针对的目标（例如，要点击的菜单项），操作则描述了要对元素实施的动作（例如，点击、打开、最小化等）。

下方列表中显示了上方列表中当前所选导航操作的一系列输入项。例如，如果要执行的操作是点击，则可能需要为这个点击操作提供坐标。这个列表是动态的，取决于在当前行中所选的操作。

添加新的导航操作

请先点击“添加”按钮，以添加新的行。然后，从左侧的应用程序资源管理器中将应用程序元素拖放到相应的字段中。

根据应用程序元素的性质（尤其是您在创建应用程序元素时是否选择使其成为动态元素），您可能需要通过点击相应的按钮，来为应用程序元素提供参数。这样，在流程中运行业务对象时，就能识别相应的元素。

接着，您应该从下拉列表中选择操作。这个列表会根据所选应用程序元素的类型发生动态变化。如果某个操作未出现在这个列表中，则意味着无法针对所选应用程序元素执行这个操作。

最后一个字段会指明所选操作是否有任何输入项（要在下方列表中填写的项目），以及您是否已提供这些输入项。

每一步后暂停

您可以进行相关设置，以确保在每完成导航环节中所配置的一个导航操作后都会暂停运行。这可通过在表单的“每一步后暂停”字段中设置相应的表达式来实现。如果没有设置，则运行时引擎不会在每完成环节中的一个步骤后就暂停。

该字段中的表达式的求值结果必须是一个数字（表示要暂停的秒数）或时间间隔（表示要暂停的时长）。如果提供的是数字，那么对于小数部分，运行时引擎也会严格遵守。负值会被视为 0—即，不会暂停。
