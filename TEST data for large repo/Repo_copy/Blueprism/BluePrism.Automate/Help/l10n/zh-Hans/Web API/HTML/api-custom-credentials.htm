自定义凭据

此示例演示了如何将安全存储在凭据中的 API 密钥作为查询字符串传递到请求 URL 中。

配置凭据

API 密钥存储在凭据的密码字段中。

配置 Web API 定义

在 API 定义的“通用身份验证”屏幕中选择自定义身份验证类型。由于未在请求标头中传递凭据，因此无需选择凭据，而是选择“向流程公开”以创建将确定流程中所需凭据的输入参数。

在请求的 URL 中，输入 [Credential.Password] 来代替 API 密钥。

这会将给定凭据的密码字段参数化，从而允许传递该值。可以使用相同的语法传递凭据中的其他值：

  • [Credential.Password]
  • [Credential.UserName]
  • [Credential.AdditionalProperties<propertyname>]

在对象或流程中指定凭据

在流程操作中选择所需的业务对象和操作，并且可将 API 定义中设置的参数用作输入参数。参数的值是所需凭据的名称（放在引号中）。

运行对象或流程时，凭据中的 API 密钥将用在请求中：

GET http://www.postcodeuk.com/address/UK/"Andy Taylor"?api_key=FGRS5-PUUDW-NBC2Q-96UYR-QBDSY
