Blue Prismリリースマネージャー

［パッケージを変更］ウィザード

［パッケージを変更］ウィザード

概要

［パッケージを変更］ウィザードは、既存のパッケージを変更するときに使用します。パッケージでカスタマイズ可能な要素（名前、説明、コンテンツ）は変更できます。

パッケージを変更しても、そのパッケージから作成されたリリースには影響はありません。

［パッケージを変更］ウィザードの機能と使用方法は、［新しいパッケージ］ウィザードとほぼ同じです。ただし、ウィンドウのタイトルと、ステージにパッケージの現在の値があらかじめ入力されている点が異なります。

このウィザードを起動するには、リリースマネージャーに表示されるタスクリストの［パッケージを変更］タスクを使用するか、［パッケージの詳細］パネルで［パッケージを編集］ボタンを押します。

ステージ

パッケージの変更に関与するステージは次のようになります。

名前
パッケージの名前を変更する
説明
パッケージの説明を変更します
コンテンツ
パッケージのコンテンツを変更します

名前

パッケージの名前をここで入力します。パッケージ名は一意でなくてはなりません。

パッケージには名前を付ける必要があります。1文字から255文字の範囲で入力します。

説明

パッケージの説明をここに入力します。記録用として使うものであり、パッケージが選択されると［パッケージの詳細］パネルに表示されます。

必須ではなく、文字数の制限はありません。

内容

パッケージを構成するコンポーネントは、ここで決定されます。これは2つのパネルに分割されています。左側のペインには、データベースのすべてのコンポーネントが表示され、右側のペインにはパッケージのコンテンツが表示されます。

コンポーネントをパッケージに追加するには、コンポーネントを左側のペインで探し、それを右側のペインにドラッグします。

コンポーネントをパッケージから削除するには、手順を逆に実行します。つまり、右側のペインから左側のペインにコンポーネントをドラッグして戻します。

どちらのペインでも同じタイプの複数のコンポーネントを選択して、もう片方のペインにドラッグできます。

コンポーネントは、ドラッグ元のペインから削除されます。たとえば、プロセスP1を環境からパッケージにドラッグした場合、P1は環境ペインから削除され、パッケージペインに追加されます。

環境のグループからパッケージにプロセスをドラッグした場合、グループもパッケージに暗黙的に追加されます。

パッケージにスケジュールが追加されると、依存関係が確認され、リリース作成時に追加されます。たとえば、特定のカレンダーが必要な場合、パッケージ作成時ではなくリリース時にそのカレンダーが特定され、含まれます。

完了

コンテンツ ステージで［次へ>］をクリックすると、ウィザードによってデータベースにパッケージが作成されます。これが完了したら、ウィザードを閉じます。

エラーが発生した場合、メッセージが表示され、ユーザーは前のステージに戻ることを選択し、パッケージに変更を加え、再試行できます。
