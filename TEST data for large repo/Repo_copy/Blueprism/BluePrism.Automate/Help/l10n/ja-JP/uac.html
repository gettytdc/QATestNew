Blue Prism構成

Windows VistaまたはWindows 7のユーザーアカウントコントロール

概要

Windows Vistaは、Windows環境に「ユーザーアカウントコントロール」を導入しました。この導入は、すべてのユーザーに影響するシステムへの変更を、そのような変更が行われたときに昇格した権限を要求することでコントロールできるメカニズムです。

このメカニズムがBlue Prismソフトウェアに与える影響

Blue Prismの観点からは、以下の構成変更のいずれかに影響します。

  • クライアントPCとリソースPCが接続するデータベースやサーバー接続のリスト。
  • マシン上でBlue Prismクライアントアプリケーションが起動したときにリソースマシンが実行されるかどうか
  • マシンに登録されている外部のビジネスオブジェクト
  • マシンに定義されたBlue Prismサーバー構成

Blue Prismが管理者として実行されている（つまり、プログラムが常に 「管理者として実行」 するように設定されている）場合は、ユーザーからの確認を求めずに設定を保存します。

アプリケーションが通常の権限で実行されている場合、ユーザーには最初にマシン構成を保存する必要があるときに、昇格のメッセージが表示されます。受け入れられると、AutomateConfigという別のプログラムが実行され、Blue Prismクライアントを実行している限りAutomateConfigは引き続き実行されます。

AutomateConfigが実行されていることは通知領域（画面の右下）で確認するとわかります。小さな歯車が上に重ねられたBlue Prismアイコンがあります。

プログラムが必要以上に開いたままになっている場合は、アイコンを右クリックして［終了］を選択すると、プログラムを閉じることができます。
