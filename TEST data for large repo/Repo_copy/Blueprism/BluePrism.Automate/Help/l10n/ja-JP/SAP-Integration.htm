ヘルプ - SAP統合

概要

SAP統合は、SAP GUIスクリプトAPIを使用してBlue Prismにより実行されます。このAPIはSAP AGによって提供されており、シッククライアントプラットフォームであるNetweaverを自動化できます。

使用法

Blue PrismがSAPと対話する前に、GUIスクリプトAPIをSAPサーバー上で有効にする必要があります。SAP GUIスクリプトを有効にするには、SAPクライアントを開き、管理者の認証情報を使用してログオンします。RZ11トランザクションを開始します。［プロファイルパラメーターの更新］ウィンドウに 「sapgui/user_scripting」 と入力して［表示］をクリックします。［プロファイルパラメーター属性の表示］ウィンドウで［値を変更］をクリックし、［新しい値］フィールドに「TRUE」と入力します。 設定を保存してSAP GUIからログアウトします。

サポートされているコントロール

SAPボタン

ボタンコントロールは1つのナビゲートアクション 「押す」 をサポートします。境界とテキストは読み取りアクションを使用して取得できます。このコントロールには書き込めません。

SAPチェックボックス

標準のチェックボックスコントロールは値の読み書きをサポートします。

SAPコンボボックス

コンボボックスアイテムは、「アイテムを選択」 アクションを使用してインデックスまたは名前で選択できます。 アイテム数は読み取りステージを使用して読み取ることができます。選択したテキストを読み取ることができ、コンボボックス内のアイテムの全リストは ［すべてのアイテムを取得］を使用して読み取ることができます。

SAPグリッド表示

グリッドビューは、グリッドのコンテンツ全体のコレクションへの読み込みをサポートしています。テキストベースの値のみが表示されます。アイコンなどを含む列は空になります。

SAPラベル

ラベルコントロールは、読み取りステージを介したテキストの取得をサポートしています。

SAPメニューバー

SAP GUIスクリプトAPIの予測不能な点は、メインのSAPウィンドウが最大化されたときにのみスパイを実行できるという点にあり、最大化された状態かどうかに関わりなく、一度スパイできれば正常に機能します。他のアプリケーションタイプと同様に、パスによるメニューアイテムの選択をサポートしています。パスの例は "{File}{Save}" です。

SAPスクロールバー

SAPスクロールバーは、読み取りステージで最大値を取得、値を取得、最小値を取得、値を設定ナビゲートアクション、値を設定する書き込みもサポートしています。

SAPスクロールコンテナ

SAPスクロールバーコンテナは、SAPスクロールバーと同様の一連のアクションでサポートされています。 違いは、それぞれのアクションが水平方向と垂直方向に行われ、書き込みステージ機能がないことです。

OKコードフィールド

OKコードフィールドはツールバーにある特別なコンボまたはテキスト入力ボックスで、「SU11」のようなトランザクションコードを入力できます。コードを入力すると、ユーザーはアプリケーション内の宛先に移動します。書き込みステージを使用した、OKコードフィールドへのテキストの設定がサポートされています。 これをアクティブにするには、Enterキーを押すのをシミュレートする必要があります。

SAPパスワードフィールド

SAPパスワードフィールドは、書き込みステージを使用して書き込むことができます。

SAPラジオボタン

SAPラジオボタンでは書き込みと選択を実行できます。テキストと現在の状態も読み取ることができます。 書き込み時には、値をFalseに設定できません。これによりラジオの設定が壊れためです。 値をFalseに設定する場合、例外が発生します。

SAPテーブルコントロール

テーブルコントロールは、テーブルのコンテンツ全体のコレクションへの読み込みをサポートしています。 テキストベースの値のみが表示されます。アイコンなどを含む列は空になります。

SAPタブ

SAPタブコントロールは、タブストリップ内の個々のタブを表します。タブは直接スパイでき、［選択］アクションを使用してタブを選択できます。

SAPタブストリップ

タブストリップコントロールは、タブグループの親です。直接タブをスパイすることはできませんが、モデルからは選択できます。名前でアイテムを選択すると実行されます。

SAPテキストフィールド

標準の編集フィールドです。読み書きがサポートされています。

SAP Cテキストフィールド

Cテキストフィールドコントロールは、最近使用したエントリにアクセスするためのコンボボタンが追加されている点を除いて、テキストフィールドと同じです。

SAPツリー表示

SAPツリービューは、選択されたノードの 「キー」 の読み取り、特定のキーのノードテキストの取得もサポートしています。これら二つを順番に組み合わせると、選択したノードテキストを取得できることになります。これは、1つの読み取りステージの2つのステップとして実行できます。標準の 「アイテムを選択」 アクションは、直接のワンステップアクションを実行します。 他のアプリケーションタイプのツリービューとは異なり、SAPツリーノードにはチェックボックスを付けるできません。したがって、「アイテムに選択済みの値を設定」 はSAPツリービューには実装されていません。SAPのツリーノードの展開状態または折りたたみ状態を判断することは不可能であるため、「アイテム展開を切り替える」 の実装、その状態の読み取り方法の提供、または展開機能や折りたたみ機能の自動テストは実行できません。SAP APIを介して 「アイテムを見えるようにする」 は実装できません。ツリーノードを選択するときには、その親はすでに展開されています。APIは自動的に展開せず、非表示になっているとノードを見つけることはできません。

HTMLビューアー

さらに、SAP APIは基礎となるHTMLドキュメントのビューを提供し、Blue PrismはHTMLスパイモードを介してアクセスされるこれらのコントロールと統合できます。

サポートを拡大する

SAPコネクターは、テキストベースの構成ファイルを使用して設定できます。これらの構成ファイルの変更方法の説明は、このヘルプドキュメントには記載されていません。構成ファイルの使用に関しては、Blue Prismサポートにお問い合わせください。
