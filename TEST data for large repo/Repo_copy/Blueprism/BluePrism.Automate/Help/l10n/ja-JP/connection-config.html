接続構成

概要

ログインページからアクセスできる［接続構成］フォームは、Blue Prismデータベースおよびサーバーに対して複数の接続を構成できるインターフェイスを提供します。

このソフトウェアは、SQL ServerまたはWindows認証を使用して、SQL Server 2005以降に接続できます。あるいは、利用可能なネットワーク内のサーバーで動作中のBlue Prismサーバーを経由して接続することもできます。

接続はデフォルトでマシンごとに構成されます。つまり、接続リストは現在のマシンのすべてのユーザーが利用でき、任意のユーザーが行った変更は同じマシンでBlue Prismを使用する次のユーザーに表示されます。 これをユーザーごとに接続リストを作成するように変更するには、/useroptsスイッチを指定してBlue Prismを起動します。

既存の接続を構成する

接続をフォームの左側にあるリストから選択し、右側に表示されるフィールドで詳細を編集すると構成できます。

接続名

この名前は、ログイン画面またはBPサーバー構成のいずれかで製品内の接続を説明するために使用され、Blue Prismが使用する接続を示します。

接続タイプ

Blue Prismでは、いくつかの接続（および認証）タイプがサポートされています。

SQL Server（SQL Server認証）
SQLログイン（データベースに設定されたユーザー名とパスワード）を使用して認証されたSQL Serverインスタンスへの直接接続です。
SQL Server（Windows認証）
Windowsユーザーアカウントを通じて認証されたSQL Serverインスタンスへの直接接続です。
Blue Prismサーバー
既存のBlue Prismサーバーへの接続です。サーバーはデータベース自体への接続を維持します。
可用性グループ（SQL Server認証）
SQLログイン（データベースに設定されたユーザー名とパスワード）を使用して認証されたSQL Server可用性グループへの直接接続です。
可用性グループ（Windows認証）
Windowsユーザーアカウントを通じて認証されたSQL Server可用性グループへの直接接続です。

接続タイプの相違点の詳細については、Blue Prismサーバーのヘルプトピックを参照してください。

データベースサーバー（SQL Serverのみ）

Blue Prismが接続するデータベースインスタンスです。例：localhost\SQLExpressは、ローカルマシンで動作中のSQLExpressという名前のインスタンスに接続します。

データベース名（SQL Serverのみ）

Blue Prismがデータを保管している、指定したインスタンス上のデータベースの名前です。

ユーザーID（SQL Server - SQL認証のみ）

Blue PrismがSQL Serverへの認証に使用するSQLユーザーのIDです。

パスワード（SQL Server - SQL認証のみ）

Blue PrismがSQL Serverへの認証に使用するパスワードです。

追加のSQL接続パラメーター（BP以外のサーバーのみ）

データベースにアクセスするために作成したクエリ文字列に追加する必要がある、追加のセミコロン区切りのパラメーターです。例：Encrypt=true;TrustServerCertificate=true

マルチサブネットフェイルオーバー

接続でマルチサブネットフェイルオーバーを有効にできます。詳細については、これに関するMicrosoftのドキュメントを参照してください。

Blue Prismサーバー（BPサーバーのみ）

クライアントが接続する必要があるBlue Prismサーバーのネットワークアドレスです。

接続モード（BPサーバーのみ）

以下の接続モードは、Blue PrismクライアントとBlue Prismサーバー間の接続に使用できます。この設定は、Blue Prismサーバー構成で設定した対応する値と一致する必要があります。

WCF：メッセージの暗号化付きSOAP＆Windows認証
  • デバイス間に信頼関係が必要です。はい
  • Blue Prism認証モード：Blue Prismネイティブ/シングルサインオン
  • サーバー側の証明書が必要です。いいえ
  • トランスポート：SOAP over HTTP

メッセージコンテンツのみが暗号化されます。  SOAPヘッダーとHTTPヘッダーは暗号化されないままであるため、複雑なルーティング、ロードバランサー、プロキシなどが容易になります。  クライアントとサーバーIDは、Windows/Active Directoryを介して検証されます。

WCF：転送の暗号化付きSOAP＆Windows認証
  • デバイス間に信頼関係が必要です。はい
  • Blue Prism認証モード：Blue Prismネイティブ/シングルサインオン
  • サーバー側の証明書が必要です。はい
  • トランスポート：SOAP over HTTPS

SOAPヘッダーを含むトランスポートは、証明書ベースの暗号化を使用して暗号化されます。クライアントとサーバーIDは、Windows/Active Directoryを介して検証されます。

WCF：転送の暗号化付きSOAP
  • デバイス間に信頼関係が必要です。いいえ
  • Blue Prism認証モード：Blue Prismネイティブ
  • サーバー側の証明書が必要です。はい
  • トランスポート：SOAP over HTTPS

SOAPヘッダーを含むトランスポートは、証明書ベースの暗号化を使用して暗号化されます。サーバーIDは証明書を使用して検証されます。

.NET Remoting：セキュア
  • デバイス間に信頼関係が必要です。はい
  • Blue Prism認証モード：Blue Prismネイティブ/シングルサインオン
  • サーバー側の証明書が必要です。はい
  • トランスポート：TcpChannel over SChannel

後方互換用に提供されます。暗号化はクライアントとサーバーの間でネゴシエートされます。クライアントとサーバーIDは、Windows/Active Directoryを介して検証されます。

.NET Remoting：非セキュア
  • デバイス間に信頼関係が必要です。いいえ
  • Blue Prism認証モード：Blue Prismネイティブ
  • サーバー側の証明書が必要です。いいえ
  • トランスポート：TcpChannel

推奨しません - 後方互換用に提供されます。 接続セキュリティはサードパーティのソリューションによって全面的に提供される必要があります。

WCF：非セキュア
  • デバイス間に信頼関係が必要です。はい
  • Blue Prism認証モード：Blue Prismネイティブ/シングルサインオン
  • サーバー側の証明書が必要です。はい
  • トランスポート：SOAP over HTTP

推奨しません。接続セキュリティはサードパーティのソリューションによって全面的に提供される必要があります。

サーバーポート（BPサーバーのみ）

指定されたBlue Prismサーバーが接続をリッスンしているポートです。

コールバックポート（.NET Remotingを使用するBPサーバーのみ）

.NET Remoting接続モードでBlue Prismサーバーに接続すると、クライアントはサーバーに「コールバック」を許可します。双方向通信を有効にすることで、一部のサーバー操作で進捗状況に関するよりタイムリーなフィードバックを得ることができます。この値をデフォルト（ゼロ）のままにすると、サーバーへの接続が確立されたときに任意のコールバックポートが割り当てられます。特定のポートを使用する必要がある場合は（ファイアウォールのポートを開くためなど）、ここで設定する必要があります。

特定のポートが構成されている場合、このマシンでは一度に1つのBlue Prismのインスタンスのみがこの接続を使用できます。特定のコールバックポートが使用されているため、現在のマシンからサーバーに、さらに接続しようとしても失敗します。

テスト接続

右側の［現在の接続］詳細ペインに現在表示されている接続は、［テスト接続］ボタンをクリックしてテストできます。

構成されたBlue Prismデータベースへの接続が試行され、その試行結果を含むメッセージボックスが表示されます。

新しい接続を作成する

フォームの左下にある［新しい接続］ボタンをクリックすると、新しい接続を作成できます。

詳細が空白の任意の名前の接続が作成され、詳細ペインに表示されます。

接続を削除する

選択ペインで接続を選択し、［接続を削除］ボタンを押すと、その接続を削除できます。

注意：このアクションに対する確認プロンプトはありません。

変更を保存する

接続が正しく構成されたら、［OK］ボタンを押して保存できます。

これにより、このマシン上のすべてのユーザーの接続構成が保存され（Blue Prismが/useroptsコマンドラインスイッチで起動していない限り）、現在選択されている接続がBlue Prismクライアントの現在の接続として設定されます。

変更を破棄する

間違いがあった場合は、［キャンセル］ボタンを押して変更を破棄できます。フォームは変更を保存せずに閉じられます。

Windows Vista/Windows 7のユーザーアカウントコントロール

Windows Vista以降で接続構成を保存する場合は、特権の昇格が必要になることがあります。詳細については、「ユーザーアカウントコントロール」ヘルプページを参照してください。
