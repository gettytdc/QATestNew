Java Automation

此功能受许可限制。有关详细信息，请参阅许可页面。

概述

Java Automation 可提供一系列专用技术，以便与使用 Java 编程语言编写的应用程序进行交互。要充分利用这些功能，您应该在运行应用程序建模器向导时指明您的应用程序属于 Java 类型。或者，如果在应用程序建模器向导中选择了相应的应用程序参数，则也可在浏览器应用程序中使用 Java 集成技术。

要求与安装

要使用 Java Automation，就需要安装 Java Access Bridge。在撰写本文时，最新的独立版本为 2.0.2。近期的 JRE 版本都会将 Access Bridge 作为标准配置随附提供。

Blue Prism 支持 JAB 2.0.0 版以及 2.0.2 版和更高版本。2.0.1 版存在会导致 Blue Prism 出现问题的缺陷，所以请避免使用这个版本。

请参考 Java Access Bridge 安装指南，以详细了解如何安装 Java Access Bridge 并与 Blue Prism 搭配使用。

使用

您可以在应用程序建模器中使用探测工具像访问 Object Studio 中的任何其他应用程序一样访问 Java 应用程序。 请参阅探测工具返回的 java 属性列表，以进一步了解如何识别目标应用程序中的元素。

  • 使用 32 位操作系统的客户可以使用 Java Access Bridge 2.0.0 或更高版本，并可将应用程序设置为以嵌入式或外部 32 位模式启动。
  • 想要使用 64 位操作系统的客户则必须使用 Java Access Bridge 2.0.2 或更高版本。
  • 必须将用于对 64 位 Java 应用程序建模的业务对象设置为“外部 64 位”模式。

限制

Sun 的 Java Access Bridge 只能为 Java Runtime Environment (JRE) 1.3.x 版提供有限支持。尤其是，无法在编辑字段中直接编写文本。但是，每项限制都存在相应的变通方案，这意味着 Java Automation 对于 JRE 1.3 而言仍是一个非常有用的工具。 请另行参阅帮助主题 32808。

Java Access Bridge 不允许与模态对话框进行交互。这一点在 Sun 的邮件列表中有所记述。 但是，Blue Prism 提供了多种其他工具以用于与模态对话框进行交互，包括位图识别、字体识别、针对基本用户交互使用区域概念（如点击鼠标）。
