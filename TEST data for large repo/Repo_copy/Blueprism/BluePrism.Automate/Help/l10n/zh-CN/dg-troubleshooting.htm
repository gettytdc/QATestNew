故障排除 数据网关

我如何检查数据网关系统是否按预期方式工作？

可通过多种方式检查数据网关的问题：

  • 在控制室中查看数据网关流程的状态

    如果它处于错误状态，状态消息将提供与问题相关的信息。

  • 在应用程序服务器上查看事件日志。

    任何数据网关流程错误都将从 Logstash 流程管理器源输出到事件日志中。Logstash 流程的标准输出流也会在发生错误时写入事件日志，这有助于识别 Logstash 相关错误。

  • 在应用程序服务器上查看 Logstash 日志。

    这些日志会写入 C:\Logstash\logstash\logs\。

我是否应对数据网关使用专用的应用程序服务器？

对大多数安装来说，可以对数据网关使用现有的应用程序服务器。但是，在数据网关用来处理大型数据集或将数据导向多个目标的情况下，将数据网关部署在专门用于此用途的 Blue Prism 应用程序服务器上可能很合适。

我是否可以更改数据网关流程用来连接到 SQL Server 的端口？

默认情况下，数据网关流程使用端口 1433。但是，可以通过编辑自定义配置来指定其他端口。

端口在配置文件结构的 Input 部分中定义，如下面的示例所示。编辑自定义配置并用所需端口号替换突出显示的值：

input {

jdbc {

jdbc_driver_library => "..\sqljdbc_4.2\enu\jre8\sqljdbc42.jar"

jdbc_connection_string => "jdbc:sqlserver://SQL_SERVER_INSTANCE:1433;databaseName=BP_

DATABASE;"

jdbc_driver_class => "com.microsoft.sqlserver.jdbc.SQLServerDriver"

jdbc_user => "<%Data Gateways BP Database SQL User.username%>"

jdbc_password => "<%Data Gateways BP Database SQL User.password%>"

statement => "delete top(3000)from BPADataPipelineInput with (rowlock, readpast)

output deleted.eventdata"

schedule => "*/3* * * * *"

}}
