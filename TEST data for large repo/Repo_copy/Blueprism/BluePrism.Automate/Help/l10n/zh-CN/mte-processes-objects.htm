多组环境

流程和对象的访问权限

流程和对象的访问权限分为两种类型，即项目和组。

项目权限

这些权限适用于组中的流程或对象，并决定了用户可以对这些项目执行哪些操作。

权限

描述

删除

用户可以删除组中的项目。

编辑

用户可以编辑组中的项目。假定还具有执行和查看权限。

执行

用户可以执行项目，但无法打开或编辑它们。执行权限不会自动授予查看定义的权限，这需要“编辑”权限。具有“执行”权限但没有“查看定义”或“编辑”权限的用户在调试对象或流程时无法步入该项目。

导出

如果用户具有相关访问权限，则可以导出流程、对象和发行版。

查看定义

用户可以查看对象或流程，但无法运行或编辑它。这还可以防止用户成功运行引用其只具有查看定义权限的对象的流程。如果用户具有查看和运行流程的权限，则只有在到达禁止对象时才会停止运行。

作为 Web 服务执行

用户可以调用作为 Web 服务呈现的对象或流程

组权限

这些权限决定了用户可以执行的与组相关的操作。

权限

描述

创建

用户可以在组中创建项，包括在编辑器中使用“另存为”功能。假定还具有“编辑”、“执行”和“查看”权限。

编辑组

用户可以在 Studio 树中创建、重命名、移动和删除组。如果用户也具有“管理访问权限”的权限，则还可以撤回和取消撤回流程。

移动操作中涉及的任何受限制的组都需要“管理访问权限”和“编辑组”的权限。如需更多信息，请参阅移动组。

导入

如果用户的用户角色具有相关访问权限，则可以从“文件”菜单或“发行版”选项卡导入项目。

管理访问权限

用户可以编辑有权访问组的角色的访问权限，以细化角色允许的权限。具有此权限的用户不能授予为用户角色设置的权限拒绝的访问权限

用户不能为其自身的用户角色更新此权限。这可以防止用户授予自己之前被拒绝访问的组访问权限管理权限。
