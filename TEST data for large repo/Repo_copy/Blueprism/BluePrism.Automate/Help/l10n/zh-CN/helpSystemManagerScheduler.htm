调度程序配置

概述

Blue Prism 调度程序作为 Blue Prism 服务器服务的一部分运行，但在产品中基于每个连接进行配置。 默认情况下，调度程序不会在新连接上运行。要利用它，必须在系统管理器的“系统”部分将其设置为启用。

任何启用了“系统管理器—调度程序”首选项的用户都可以为某个连接配置调度程序。

激活调度程序

通过勾选“为此环境激活调度程序”框，可以启用调度程序。

请注意，仅在启动 BP 服务器服务时才会检查此设置，因此必须在更改此值后重新启动它以启用或禁用调度程序。

配置调度程序

要启用其他调度程序配置，必须先激活调度程序。

检查错过的计划

初始化调度程序时，如果调度程序未运行，可能会重新检查最近应执行的计划。

要配置调度程序应重新检查的时间范围，可在“执行过去 n 分钟内错过的计划”文本框内输入要重新检查的分钟数。

要将调度程序设置为不重新检查，应输入值 0（零）。

请注意，如果发现应在配置的期间内执行多次的计划，则仅执行一次。同样，已成功运行的计划将不会再次执行。

另请注意，此设置仅在调度程序启动时有效，因此它对正在运行的调度程序没有任何影响。

设置离线资源的重试周期

如果调度程序尝试在某个资源上运行流程时，发现该资源处于离线状态，则会等待并在定义的秒数之后重试资源。

通过在“n 秒后重试”文本框中输入一个数字，可配置重试之间等待的秒数。

设置离线资源的重试次数

调度程序会多次测试离线资源，以查看它是否变为可用。

可以通过在“重试最多 n 次”文本框中输入一个数字来配置重试资源的次数。

要将调度程序配置为根本不重试离线资源，应在此处输入值 0（零）。
